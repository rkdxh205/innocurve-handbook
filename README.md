### Day 2 — 텔레그램 뉴스봇, 오늘 하루 안에 처음부터 끝까지 완성 (확정)
Day 2는 "내 이름 + 관심 키워드 뉴스 알림 봇"을 기획부터 실제 자동 실행까지, 하루 만에 전부 완성하는 것을 목표로 합니다.

1. **Telegram Desktop 설치 & 가입** — 노트북만으로 진행하며, 스마트폰 앱 설치는 필요 없습니다. 다만 계정 가입 시 인증 코드를 문자(SMS)로 받아야 하므로 본인 명의 휴대폰 번호는 그대로 필요합니다.
2. **GitHub 계정 준비** — 이후 자동화 단계에서 코드를 올릴 저장소용 계정을 미리 만들어 둡니다.
3. **BotFather로 봇 생성 & 봇 토큰 발급** — `/newbot` 명령으로 나만의 봇을 만들고 username·토큰을 발급받습니다.
4. **Telegram Bot API 기초** — `sendMessage`로 메시지 보내기, `getUpdates`로 메시지 확인하기, 내 `chat_id` 알아내기.
5. **네이버 오픈 API(뉴스검색) 발급** — 개발자센터에서 애플리케이션을 등록하고 Client ID·Secret을 발급받습니다.
6. **실전 뉴스 수집** — 이름 조합 검색어와 관심 키워드 검색어를 각각 API로 호출하고, `RECENT_HOURS` 값으로 몇 시간 이내 기사까지 받을지(실시간성) 직접 정합니다.
7. **필터링** — 동명이인은 필수·제외 키워드 조합으로, 관심 키워드 기사는 포함 여부로 판단해 진짜 원하는 기사만 남깁니다.
8. **중복 발송 방지** — `seen_urls`에 전송한 기사 URL을 저장해 같은 기사를 두 번 보내지 않도록 하고, 추적 파라미터를 제거해 정규화합니다.
9. **완성 — 실행해서 전송하기** — 검색 → 시간 필터 → 내용 필터 → 중복 제거 → 전송의 전체 로직을 하나로 합쳐 로컬에서 직접 실행하고 동작을 확인합니다.
10. **GitHub Actions로 자동 실행** — 코드를 GitHub 저장소에 올리고 Secrets를 등록한 뒤, cron 스케줄로 하루 2회(아침·저녁) 자동 실행되도록 배포합니다. 실행 결과(`seen_urls`)는 워크플로가 자동으로 저장소에 다시 커밋합니다.

각 단계는 **매뉴얼**(이론·개념 설명)과 **실습 예시**(초급 2종·중급 2종·고급 3종, 총 7개 시나리오)에 모두 반영되어 있어, 읽으면서 바로 따라 만들 수 있습니다.
