# 바이브 코딩 가이드북 — 나만의 AI 앱 만들기 (4일 실습 가이드북)

코딩 경험이 전혀 없어도 4일이면 충분합니다. 생성형 AI의 기초와 개발 환경 구축부터 시작해, Claude와 함께 자연어로 **나만의 AI 앱**을 기획(PRD)·설계(PLAN·DESIGN)·구현·검증·개선해 첫 PDCA 한 바퀴를 완성하고, Vercel로 실제 배포까지 처음부터 끝까지 직접 완성하는 개인 실습형 가이드북입니다. 과정 전체는 **문서 Q&A 챗봇을 예시**로 따라가며, 각자 자신의 앱 아이디어에 그대로 적용할 수 있습니다.

## Day별 상세 구성

### Day 1 — AI 개념과 개발 환경 구축 (확정)
생성형 AI·LLM 개념과 최신 모델 동향부터 실습에 필요한 도구까지 개념을 잡고, 자연어 프롬프트로 첫 웹페이지를 만들어 배포합니다.
- 생성형 AI·LLM이란 무엇이고 어떻게 쓰는가
- 실습 도구 개념 정리 — Node.js · Git · GitHub · Vercel · API Key · Claude Desktop · 바이브 코딩 · 배포
- STEP 1~5 — Node.js·Git 설치, 계정 준비, Claude Desktop 설치
- 자연어 프롬프트로 헬로페이지를 만들어 수정하고 Vercel로 배포하기

### Day 2 — AI 애플리케이션 개념과 PRD 작성 (확정)
만들기 전에 먼저 계획합니다. 문서 Q&A 챗봇(RAG)의 개념을 체험하고, 내가 만들 앱의 요구사항을 문서로 정리합니다.
- 문서 Q&A 챗봇(RAG)의 개념
- PRD(제품요구사항정의서)의 필요성과 구성 요소
- Claude Desktop으로 RAG 직접 체험
- 내 앱의 PRD를 작성하고 AI로 검토·구체화해 `PRD.md` 한 편 완성 (문서 Q&A 챗봇을 예로)

### Day 3 — Claude Code 심화와 PDCA (확정)
Claude Code를 깊이 있게 다루고, PDCA 방법론으로 앞서 만든 PRD를 계획·설계로 옮깁니다.
- 스킬·플러그인 활용
- 외부 도구·데이터 연결 — MCP
- 하네스 엔지니어링 — 반복 가능하고 안정적인 결과를 위한 규칙·작업·절차·검증(`CLAUDE.md`)
- PDCA 방법론과 Plan·Design 개념
- Day 2의 PRD를 `PLAN.md`·`DESIGN.md`로 계획·설계

### Day 4 — bkit PDCA 완성과 배포·발표 (확정)
PDCA의 나머지 세 단계로 실제 앱을 완성하고, 배포·발표·수료로 마무리합니다.
- Do — 실제 기능 구현
- Check — Gap 분석과 설계 대비 검증
- Act — 오류 수정·개선 반복·기능 고도화
- Day 3의 `PLAN.md`·`DESIGN.md`대로 내 앱을 완성해 첫 PDCA 한 바퀴 마무리
- 결과물 Vercel 배포 및 발표 준비
- 결과물 발표·시연, 전문가 피드백 및 종합 평가·수료
