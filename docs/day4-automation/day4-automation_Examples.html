● 초급 Getting Started Day 2~3에서 작성한 코드를 처음으로 GitHub에 올려보는 단계
G-1 초급 새 GitHub 저장소를 만들고 지금까지 작성한 코드 올리기
Day 2~3에서 Claude Code와 함께 완성한 뉴스 알림 봇 코드가 지금은 내 컴퓨터 안에만 저장되어 있습니다. 이 코드를 GitHub Actions로 자동 실행하려면 먼저 GitHub 저장소에 올려두어야 합니다. 웹 화면으로 새 저장소를 만들고, 터미널 명령어는 몰라도 Claude Code에게 자연어로 요청해 코드를 업로드합니다.
사용 요소
New repository Public / Private
단계별 실행
  1. 웹 화면에서 새 저장소 만들기

    github.com에 로그인한 뒤, 우측 상단 + 아이콘 → New repository를 클릭합니다. Repository name에 news-alert-bot 같은 이름을 입력하고, 공개 범위는 Private를 선택한 뒤 Create repository를 클릭합니다.

    Repository name: news-alert-bot
Visibility: Private
→ Create repository 클릭 후 생성되는 주소 예시
https://github.com/<내-계정명>/news-alert-bot
  2. Claude Code에게 업로드 요청하기

    새로 만든 저장소 주소를 알려주면서, Claude Code에게 지금까지 작성한 코드를 그 저장소에 올려달라고 자연어로 요청합니다. Claude Code가 필요한 git 명령을 대신 실행합니다.

    방금 만든 https://github.com/내계정명/news-alert-bot 저장소에
지금까지 작성한 뉴스 알림 봇 코드를 올려줘.
Claude Code는 이 요청을 받으면 내부적으로 git init(코드 폴더를 저장소로 초기화) → git add .(변경된 파일 모두 담기) → git commit -m "..."(담은 내용을 하나의 기록으로 저장) → git remote add origin <주소>(어디로 올릴지 연결) → git push -u origin main(실제로 GitHub에 전송) 순서로 명령을 실행합니다. 완료되면 GitHub 저장소 화면에서 방금 올라간 파일 목록을 바로 확인할 수 있습니다.
코드를 이미 한 번 올린 뒤에는 다시 저장소를 만들 필요 없이, 수정한 내용을 "다시 GitHub에 올려줘"라고만 요청해도 Claude Code가 변경된 부분만 커밋하고 올려줍니다.
● 중급 Daily Workflow 비밀값을 등록하고 워크플로우를 손으로 한 번 실행해 확인하는 단계
D-1 중급 Secrets 3개 등록하고 workflow_dispatch로 수동 실행 테스트하기
코드는 이제 GitHub 저장소에 올라가 있습니다. GitHub Actions가 봇을 실행하려면 텔레그램 토큰과 네이버 API 키가 필요한데, 이 값들을 코드에 직접 적지 않고 Secrets에 등록해 둡니다. 그리고 예약 시간을 기다리지 않고도 바로 실행해볼 수 있도록 수동 실행 버튼만 있는 간단한 워크플로우 파일을 먼저 만들어 테스트합니다.
사용 요소
New repository secret workflow_dispatch
단계별 실행
  1. Secrets 3개 등록하기

    저장소 화면에서 Settings → Secrets and variables → Actions → New repository secret을 클릭해, 아래 3개를 하나씩 등록합니다. Secret란에는 실제 값을 입력합니다.

    Name: BOT_TOKEN              → Secret: (Day 2에서 발급받은 텔레그램 봇 토큰 값)
Name: NAVER_CLIENT_ID         → Secret: (Day 3에서 발급받은 네이버 Client ID 값)
Name: NAVER_CLIENT_SECRET     → Secret: (Day 3에서 발급받은 네이버 Client Secret 값)
  2. 수동 실행용 워크플로우 파일 만들기

    Claude Code에게 .github/workflows/manual-test.yml 파일을 만들어 달라고 요청합니다. 아래 내용처럼 workflow_dispatch만 있는 가장 단순한 형태로 시작합니다.

    name: 뉴스 봇 수동 테스트

on:
  workflow_dispatch:

jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.12'
      - run: pip install requests
      - name: 뉴스 체크 및 전송
        env:
          BOT_TOKEN: ${{ secrets.BOT_TOKEN }}
          NAVER_CLIENT_ID: ${{ secrets.NAVER_CLIENT_ID }}
          NAVER_CLIENT_SECRET: ${{ secrets.NAVER_CLIENT_SECRET }}
        run: python check_and_send.py
  3. 수동으로 한 번 실행해보기

    파일을 커밋해서 저장소에 올린 뒤, 저장소의 Actions 탭으로 이동합니다. 좌측에서 방금 만든 워크플로우 이름을 선택하면 Run workflow 버튼이 보이고, 이를 클릭하면 즉시 실행이 시작됩니다.

    Actions 탭 → 뉴스 봇 수동 테스트 → Run workflow 버튼 클릭
→ 실행 목록에 새 항목이 생기고, 초록 체크가 뜨면 성공
Run workflow를 누르면 GitHub가 임시 서버(ubuntu-latest 가상 머신)를 하나 띄우고, 그 위에서 코드를 내려받아 Python을 설치한 뒤 check_and_send.py를 실행합니다. 이때 env에 적어 둔 ${{ secrets.BOT_TOKEN }} 등이 실제 값으로 치환되어 스크립트에 전달됩니다.
Secret 이름과 코드 안에서 읽어오는 환경 변수 이름이 정확히 일치해야 합니다. 하나라도 철자가 다르면 실행은 성공한 것처럼 보여도 텔레그램 메시지가 오지 않을 수 있으니, 실패 시 Actions 탭의 로그에서 어떤 환경 변수가 비어 있는지 확인하세요.
● 고급 Power User 하루 2회 자동 실행과 결과 자동 커밋까지 포함한 완전 자동화
P-1 고급 cron으로 하루 2회 자동 실행 + 실행 결과 자동 커밋까지 완성하기
수동 실행 테스트까지 마쳤으니, 이제 사람이 버튼을 누르지 않아도 매일 오전 8시와 오후 6시(한국시간)에 스스로 실행되도록 완성합니다. 또한 한 번 보낸 기사를 중복으로 다시 보내지 않도록, 실행이 끝날 때마다 "이미 보낸 기사 목록"(seen_urls.txt)을 저장소에 다시 커밋하는 단계까지 포함한 완전한 워크플로우를 작성합니다.
사용 요소
on.schedule.cron Actions 탭 로그
단계별 실행
  1. 완전한 워크플로우 파일 작성하기

    Claude Code에게 .github/workflows/news-check.yml 파일을 아래 내용으로 작성(또는 D-1에서 만든 파일을 수정)해 달라고 요청합니다. cron 2개(오전 8시/오후 6시 KST를 UTC로 환산한 값), workflow_dispatch, 그리고 실행 결과를 커밋하는 단계까지 모두 포함되어 있습니다.

    name: 뉴스 알림 봇

on:
  schedule:
    - cron: '0 23 * * *'   # 매일 오전 8시 KST (UTC 23:00)
    - cron: '0 9 * * *'    # 매일 오후 6시 KST (UTC 09:00)
  workflow_dispatch:

concurrency:
  group: news-check
  cancel-in-progress: false

jobs:
  check:
    runs-on: ubuntu-latest
    permissions:
      contents: write
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.12'
      - run: pip install requests
      - name: 뉴스 체크 및 전송
        env:
          BOT_TOKEN: ${{ secrets.BOT_TOKEN }}
          NAVER_CLIENT_ID: ${{ secrets.NAVER_CLIENT_ID }}
          NAVER_CLIENT_SECRET: ${{ secrets.NAVER_CLIENT_SECRET }}
        run: python check_and_send.py
      - name: 변경사항 커밋
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add seen_urls.txt
          git diff --cached --quiet || git commit -m "Update seen_urls"
          git push
  2. 커밋 후 첫 자동 실행 기다리거나 수동으로 확인하기

    파일을 저장소에 커밋하면, 다음 예약 시각(오전 8시 또는 오후 6시)에 자동으로 실행이 시작됩니다. 바로 확인하고 싶다면 workflow_dispatch가 함께 있으므로 Actions 탭에서 Run workflow로 즉시 실행해볼 수도 있습니다.

    Actions 탭 → 뉴스 알림 봇 → 실행 목록에서 "scheduled"로 표시된 항목 확인
(수동 확인 시) Run workflow 버튼으로 즉시 실행
  3. seen_urls.txt가 자동으로 갱신되는지 확인하기

    실행이 끝난 뒤 저장소 파일 목록에서 seen_urls.txt의 최근 커밋 시각을 확인합니다. 워크플로우가 새 기사를 발견해 전송했다면, 그 URL이 파일에 추가되고 "Update seen_urls"라는 커밋이 자동으로 생성되어 있어야 합니다.

    저장소 파일 목록 → seen_urls.txt → 최근 커밋 메시지: "Update seen_urls"
커밋 작성자: github-actions[bot]
permissions: contents: write가 있어야 Actions가 저장소에 다시 push할 권한을 가집니다. concurrency 설정은 이전 실행이 아직 끝나지 않았는데 다음 예약 시각이 겹쳐 실행되는 것을 막아, seen_urls.txt에 동시에 두 번 쓰다가 충돌하는 상황을 방지합니다.
cron 시간을 바꾸고 싶다면 항상 "한국시간 − 9시간 = UTC" 공식으로 환산하세요. 예를 들어 한국시간 오후 9시에 실행하고 싶다면 UTC로는 같은 날 12시이므로 cron: '0 12 * * *'가 됩니다.
