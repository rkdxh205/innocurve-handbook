1. 네이버 오픈 API 가입·등록

네이버 오픈 API — 외부 프로그램이 네이버 서비스 데이터를 요청하는 창구

네이버 오픈 API는 네이버가 보유한 검색·번역·지도 등의 기능을 외부 프로그램에서 호출할 수 있도록 열어둔 서비스입니다. 오늘 사용할 뉴스검색 API는 이 중 하나로, 특정 검색어가 포함된 뉴스 기사 목록을 JSON 형태로 돌려줍니다. 이 API를 쓰려면 먼저 네이버 개발자센터에서 "애플리케이션"을 하나 등록하고, 그 애플리케이션 전용의 인증 열쇠(Client ID·Client Secret)를 발급받아야 합니다.

애플리케이션 등록 — Client ID·Client Secret 발급 절차

아래 절차는 developers.naver.com에서 실제로 진행하는 등록 과정입니다. 한 번만 등록해두면 이후 모든 요청에서 같은 열쇠를 계속 사용합니다.

  1. 로그인: developers.naver.com에 접속해 본인의 네이버 계정으로 로그인합니다.
  2. Application 메뉴 진입: 상단 메뉴에서 Application → 애플리케이션 등록을 선택합니다.
  3. 애플리케이션 이름 입력: 40자 이내로 원하는 이름을 입력합니다(예: "뉴스 언급 알림 봇"). 이 이름은 외부에 공개되지 않으며, 관리 목적으로만 쓰입니다.
  4. 이용 API 선택: 제공 API 목록에서 검색(Search)을 체크합니다.
  5. 비로그인 오픈 API 서비스 환경 등록: 환경으로 WEB을 선택하고, 서비스 URL을 입력합니다. 실습 단계에서는 실제 서비스 도메인이 없으므로 http://localhost 계열의 주소가 흔히 허용됩니다.
  6. 등록 완료 및 확인: 등록을 마치면 애플리케이션 상세 화면에서 Client ID와 Client Secret 값을 확인할 수 있습니다.
등록 항목입력 내용비고
애플리케이션 이름 40자 이내 자유 입력 관리용 이름, 외부 비공개
이용 API 검색(Search) 뉴스검색을 포함한 통합 검색 API 카테고리
서비스 환경 WEB 비로그인 오픈 API 방식
서비스 URL 실제 서비스 도메인 또는 http://localhost 로컬 실습 단계에서는 로컬 주소도 흔히 허용됨
Client ID와 Client Secret은 비밀번호와 같은 개인 인증 정보입니다. 코드 파일에 직접 적어두지 말고, .env 파일처럼 GitHub에 올라가지 않는 별도 파일에 저장한 뒤 불러와서 사용하세요.

2. 뉴스검색 API 요청 구조

GET https://openapi.naver.com/v1/search/news.json — 요청 엔드포인트와 인증 헤더

뉴스검색 API는 GET 방식으로 호출합니다. 요청 시 발급받은 Client ID·Client Secret을 HTTP 헤더에 담아 보내야 하며, 이 헤더가 없거나 값이 틀리면 인증 오류가 발생합니다.

headers = {
    "X-Naver-Client-Id": "YOUR_NAVER_CLIENT_ID",
    "X-Naver-Client-Secret": "YOUR_NAVER_CLIENT_SECRET"
}
url = "https://openapi.naver.com/v1/search/news.json?query=검색어"

필수 헤더 X-Naver-Client-Id, X-Naver-Client-Secret 요청 방식 GET

query / display / start / sort — 검색 조건을 지정하는 파라미터

URL 뒤에 붙는 파라미터로 검색어와 결과 범위, 정렬 방식을 지정합니다. 한글 검색어는 URL에 그대로 넣으면 깨질 수 있으므로 UTF-8로 인코딩해서 전달해야 합니다.

파라미터필수 여부범위/기본값설명
query 필수 — 검색어. UTF-8로 인코딩해서 전달(예: requests.utils.quote() 사용)
display 선택 10~100 (기본 10) 한 번에 가져올 검색 결과 개수
start 선택 1~1000 (기본 1) 검색 시작 위치(페이지네이션용)
sort 선택 sim(기본) | date sim은 유사도순, date는 최신순 정렬

💡 핵심 한 줄: 뉴스 알림 봇처럼 "방금 나온 기사"를 찾는 용도라면 sort=date로 최신순 정렬을 지정하는 것이 유사도순보다 실용적입니다.

호출 한도(하루 몇 건까지 요청 가능한지)는 애플리케이션마다 다르게 안내될 수 있습니다. 정확한 한도는 애플리케이션 등록 후 개발자센터 대시보드에서 직접 확인하세요.

3. 응답 다루기

items[] — 응답 JSON에 담긴 기사 목록

요청이 성공하면 응답 JSON의 items 배열 안에 기사들이 담겨 옵니다. 배열의 각 항목(item)이 기사 한 건에 해당하며, 아래 다섯 개 필드를 주로 사용합니다.

필드설명비고
title 기사 제목 검색어 부분에 <b> 태그가 섞여 옴
originallink 기사 원문 언론사 URL 실제 기사가 처음 게재된 주소
link 네이버 뉴스 URL 또는 원문 URL 네이버 뉴스로 연동된 경우 네이버 URL
description 기사 본문 일부 요약 title과 마찬가지로 HTML 태그가 섞여 옴
pubDate 기사 발행 일시 RFC 822 형식 문자열(예: Sun, 05 Jul 2026 09:00:00 +0900)

html.unescape / re.sub / parsedate_to_datetime — 응답을 실제로 쓸 수 있게 정제하기

응답을 그대로 텔레그램 메시지로 보내면 <b> 같은 HTML 태그와 &quot; 같은 HTML 엔티티가 그대로 노출되어 읽기 불편합니다. 또한 pubDate는 문자열이므로, 실제 날짜·시간으로 변환해야 "최근 몇 시간 이내에 나온 기사만 남기기" 같은 필터링이 가능해집니다.

import html, re
from email.utils import parsedate_to_datetime

raw_title = "<b>홍길동</b> 위원, 정책 간담회 개최"
clean_title = html.unescape(re.sub(r"<[^>]+>", "", raw_title))
# → "홍길동 위원, 정책 간담회 개최"

pub_dt = parsedate_to_datetime("Sun, 05 Jul 2026 09:00:00 +0900")
# → datetime 객체로 변환되어 시간 비교·필터링에 사용 가능

이렇게 정제한 pub_dt를 현재 시각과 비교하면, "최근 N시간 이내에 발행된 기사만 알림으로 보낸다"는 필터링 로직을 만들 수 있습니다. 이 계산 로직은 Day 4에서 자동 실행 스케줄과 함께 완성합니다.

4. 동명이인 필터링 설계

동명이인 문제 — 이름만으로 검색하면 생기는 오탐

흔한 이름일수록 이름만 검색어로 넣으면 배우, 가수, 다른 지역의 동명이인 등 전혀 관련 없는 기사가 대량으로 섞여 들어옵니다. 이 문제를 해결하려면 이름 하나만 보지 말고, "이 기사가 정말 나를 가리키는지" 판단할 수 있는 추가 단서를 함께 검사해야 합니다.

REQUIRED_KEYWORDS / EXCLUDE_CONTEXTS — 포함 키워드와 제외 키워드로 이중 판별

가장 실용적인 방법은 두 종류의 키워드 목록을 함께 두는 것입니다. 첫째, 본인의 소속 기관·직함·지역구처럼 기사에 함께 등장할 가능성이 높은 포함 키워드 중 하나 이상이 있어야 통과시킵니다. 둘째, 동명이인이 주로 활동하는 분야를 나타내는 제외 키워드가 있으면 걸러냅니다.

구분예시역할
포함 키워드 소속 기관명, 직함, 지역구 이름 기사 본문에 이 중 하나라도 있어야 "진짜 나에 대한 기사"로 인정
제외 키워드 배우, 가수, 다른 지역명, 무관한 직업군 기사 본문에 이 단어가 있으면 동명이인 기사로 간주해 제외 
REQUIRED_KEYWORDS = ["소속기관", "지역구", "직함"]
EXCLUDE_CONTEXTS = ["배우", "가수", "다른지역"]

def is_target_person(text):
    if "내이름" not in text:
        return False
    if not any(k in text for k in REQUIRED_KEYWORDS):
        return False
    if any(k in text for k in EXCLUDE_CONTEXTS):
        return False
    return True
포함 키워드를 너무 좁게 잡으면 정작 필요한 기사까지 놓칠 수 있고, 제외 키워드를 너무 넓게 잡으면 반대로 필요한 기사가 걸러질 수 있습니다. 처음에는 느슨하게 시작해 실제 알림 결과를 보며 목록을 조정하는 방식을 권장합니다.

포함 키워드 최소 1개 일치 시 통과 제외 키워드 1개라도 일치 시 탈락 Day 3 결과물 is_target_person() 함수
