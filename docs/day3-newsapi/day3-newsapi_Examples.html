Day 3 네이버 뉴스 API 연동 실습 예시

● 초급 Getting Started 네이버 개발자센터에 처음 가입하고 API 키를 발급받는 단계
G-1 초급 네이버 개발자센터에서 API 키 발급받고 안전하게 보관하기
이제 막 네이버 오픈 API라는 것을 처음 들어본 상황입니다. 뉴스검색 API를 쓰려면 먼저 네이버 개발자센터에서 애플리케이션을 등록하고 나만의 Client ID·Client Secret을 발급받아야 한다는 것을 알게 되었고, 지금 그 절차를 실제로 따라 하려고 합니다.
사용 요소
애플리케이션 등록 Client ID / Client Secret
단계별 실행
  1. developers.naver.com 접속 및 로그인

    웹 브라우저에서 developers.naver.com에 접속한 뒤, 평소 쓰던 네이버 계정으로 로그인합니다.

  2. 애플리케이션 등록 메뉴로 이동

    상단 메뉴에서 Application → 애플리케이션 등록을 클릭합니다.

  3. 애플리케이션 정보 입력

    애플리케이션 이름을 40자 이내로 입력하고(예: "뉴스 언급 알림 봇"), 이용 API 목록에서 검색(Search)을 선택합니다.

  4. 서비스 환경 등록

    비로그인 오픈 API 사용 환경으로 WEB을 선택하고, 서비스 URL을 입력합니다. 아직 실제 서비스 도메인이 없다면 http://localhost 계열 주소를 입력해도 흔히 등록이 허용됩니다.

    서비스 URL 예시: http://localhost:3000
  5. Client ID / Client Secret 확인 및 보관

    등록이 완료되면 애플리케이션 상세 화면에서 Client ID와 Client Secret을 확인할 수 있습니다. 이 값을 프로젝트 폴더의 .env 파일에 아래 형식으로 저장해두세요.

    NAVER_CLIENT_ID=YOUR_NAVER_CLIENT_ID
NAVER_CLIENT_SECRET=YOUR_NAVER_CLIENT_SECRET
Client ID와 Client Secret은 "이 요청을 보낸 사람이 등록된 애플리케이션의 소유자"임을 네이버 서버가 확인하는 열쇠입니다. 이 값이 유출되면 다른 사람이 내 이름으로 등록된 애플리케이션의 호출 한도를 소모할 수 있으므로, 코드 파일이 아닌 별도 파일에 보관하는 것이 중요합니다.
.env 파일은 반드시 .gitignore에 추가해서 GitHub 저장소에 올라가지 않도록 하세요. 실수로 이미 커밋했다면 즉시 Client Secret을 재발급받는 것이 안전합니다.
● 중급 Daily Workflow 발급받은 API 키로 실제로 내 이름을 검색해보는 단계
D-1 중급 Claude Code에게 요청해서 내 이름 뉴스 검색 스크립트 만들기
G-1에서 Client ID·Client Secret 발급을 마쳤습니다. 이제 이 열쇠를 이용해 실제로 내 이름이 들어간 최신 뉴스 기사 제목을 화면에 출력하는 간단한 파이썬 스크립트를 Claude Code에게 요청해서 만들어보려고 합니다. 프로젝트 폴더와 .env 파일은 이미 준비되어 있다고 가정합니다.
사용 요소
파라미터(query·display·sort) 응답 필드(title·pubDate)
단계별 실행
  1. Claude Code에게 스크립트 작성 요청

    프로젝트 폴더를 열어둔 채로 Claude Code에 아래와 같이 요청합니다.

    "news_search.py 파일을 만들어줘.
.env에 있는 NAVER_CLIENT_ID, NAVER_CLIENT_SECRET을 python-dotenv로 불러오고,
내 이름 '홍길동'을 검색어로 네이버 뉴스검색 API(display=10, sort=date)를 호출해서
기사 제목만 최신순으로 화면에 출력하는 코드를 작성해줘."
  2. 생성된 코드 확인하기

    Claude Code가 만들어주는 코드는 대략 아래와 같은 형태를 띱니다. 실제 발급받은 값은 .env 파일에만 저장하고, 코드에는 직접 적지 않습니다.

    import os, html, re, requests
from dotenv import load_dotenv

load_dotenv()
CLIENT_ID = os.environ.get("NAVER_CLIENT_ID")
CLIENT_SECRET = os.environ.get("NAVER_CLIENT_SECRET")

headers = {
    "X-Naver-Client-Id": CLIENT_ID,
    "X-Naver-Client-Secret": CLIENT_SECRET
}
query = requests.utils.quote("홍길동")
url = f"https://openapi.naver.com/v1/search/news.json?query={query}&display=10&sort=date"

resp = requests.get(url, headers=headers, timeout=15)
for item in resp.json().get("items", []):
    title = html.unescape(re.sub(r"<[^>]+>", "", item["title"]))
    print(title, "|", item["pubDate"])
  3. 실행하고 결과 확인

    터미널에서 스크립트를 실행해 실제 뉴스 제목 목록이 최신순으로 출력되는지 확인합니다.

    python news_search.py
requests.get()이 네이버 서버에 HTTP 요청을 보내면, 서버는 헤더의 Client ID·Secret으로 신원을 확인한 뒤 검색 결과를 JSON으로 돌려줍니다. resp.json()은 이 JSON 문자열을 파이썬 딕셔너리로 바꿔주고, items 배열을 하나씩 순회하며 제목만 뽑아 출력하는 구조입니다.
실행했는데 결과가 비어 있거나 오류가 난다면 print(resp.status_code, resp.text)를 추가해 원인을 확인하세요. 401 오류는 대개 Client ID·Secret이 잘못 입력된 경우입니다. 어떤 경우에도 실제 키 값을 코드나 화면 캡처, GitHub 커밋에 노출하지 마세요.
● 고급 Power User 동명이인 기사를 걸러내고 중복 알림을 막는 단계
P-1 고급 동명이인 필터와 중복 방지 로직까지 추가한 완성형 스크립트 만들기
D-1에서 만든 스크립트를 몇 차례 실행해보니 두 가지 문제가 눈에 띄었습니다. 첫째, 같은 이름을 가진 배우나 다른 지역 인물의 기사가 함께 섞여 나옵니다. 둘째, 스크립트를 다시 실행할 때마다 이미 확인한 기사가 반복해서 출력됩니다. 이 두 문제를 동시에 해결하는 더 완성된 스크립트를 만들려고 합니다.
사용 요소
포함/제외 키워드 필터 응답 정제(html.unescape, re.sub)
단계별 실행
  1. 포함·제외 키워드 목록 정의하기

    본인의 소속 기관·직함·지역구처럼 진짜 나를 가리키는 기사에 함께 등장할 단어를 REQUIRED_KEYWORDS에, 동명이인이 주로 등장하는 분야를 EXCLUDE_CONTEXTS에 정리합니다.

    REQUIRED_KEYWORDS = ["소속기관", "지역구", "직함"]
EXCLUDE_CONTEXTS = ["배우", "가수", "다른지역"]

def is_target_person(text):
    if "홍길동" not in text:
        return False
    if not any(k in text for k in REQUIRED_KEYWORDS):
        return False
    if any(k in text for k in EXCLUDE_CONTEXTS):
        return False
    return True
  2. 이미 보낸 기사 URL을 기억하는 중복 방지 로직 추가하기

    seen_urls.txt 파일에 이전에 확인한 기사의 link 값을 한 줄씩 저장해두고, 다음 실행에서는 이 목록에 없는 새 URL만 처리하도록 합니다.

    import os

SEEN_FILE = "seen_urls.txt"

def load_seen_urls():
    if not os.path.exists(SEEN_FILE):
        return set()
    with open(SEEN_FILE, "r", encoding="utf-8") as f:
        return set(line.strip() for line in f if line.strip())

def save_seen_url(url):
    with open(SEEN_FILE, "a", encoding="utf-8") as f:
        f.write(url + "\n")
  3. 전체 로직 하나로 합치기

    검색 → 정제 → 동명이인 필터 → 중복 확인 순서로 하나의 스크립트에 연결합니다.

    import os, html, re, requests
from dotenv import load_dotenv

load_dotenv()
headers = {
    "X-Naver-Client-Id": os.environ.get("NAVER_CLIENT_ID"),
    "X-Naver-Client-Secret": os.environ.get("NAVER_CLIENT_SECRET")
}
query = requests.utils.quote("홍길동")
url = f"https://openapi.naver.com/v1/search/news.json?query={query}&display=20&sort=date"

resp = requests.get(url, headers=headers, timeout=15)
seen = load_seen_urls()
new_count = 0

for item in resp.json().get("items", []):
    title = html.unescape(re.sub(r"<[^>]+>", "", item["title"]))
    description = html.unescape(re.sub(r"<[^>]+>", "", item["description"]))
    link = item["link"]

    if link in seen:
        continue
    if not is_target_person(title + description):
        continue

    print(title, "|", link)
    save_seen_url(link)
    new_count += 1

print(f"새로 확인된 기사: {new_count}건")
세 요소가 순서대로 맞물려 동작합니다. 먼저 is_target_person()이 동명이인 기사를 걸러내고, 그다음 seen 집합과 비교해 이미 처리한 기사인지 확인합니다. 통과한 기사만 화면에 출력하면서 동시에 seen_urls.txt에 URL을 추가해, 다음 실행부터는 같은 기사가 다시 나오지 않게 됩니다.
처음에는 키워드 목록을 느슨하게 시작해서, 실제로 어떤 기사가 걸러지고 어떤 기사가 통과하는지 며칠간 지켜본 뒤 목록을 다듬는 방식이 안전합니다. 너무 엄격하게 시작하면 정작 필요한 기사까지 놓칠 수 있습니다.
함께 보기: 오늘 완성한 seen_urls.txt 기반 중복 방지 로직과 검색 스크립트는 Day 4에서 텔레그램 전송 코드와 결합되어, GitHub Actions로 매일 자동 실행되는 완전한 뉴스 알림 봇이 됩니다.
