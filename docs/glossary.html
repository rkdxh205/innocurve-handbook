용어집 · 단어 모음집

문서를 읽다 모르는 단어가 나오면 여기서 뜻을 확인하세요. 각 문서 본문에서 밑줄이 있는 빨간 글씨 용어를 누르면 이 페이지의 해당 설명으로 바로 이동합니다.

일치하는 용어가 없습니다.

AI 기본 개념
생성형 AI (Generative AI)
글·이미지·코드처럼 새로운 결과물을 만들어 내는 AI. (AI = Artificial Intelligence, 인공지능)
LLM (Large Language Model · 거대 언어 모델)
방대한 글로 학습해 사람처럼 말을 이해하고 만들어 내는 AI 모델.
프롬프트(Prompt)
AI에게 주는 지시문. 역할·구체성·제약을 담아 구체적으로 쓸수록 결과가 좋아집니다.
바이브 코딩(Vibe Coding)
문법을 직접 타이핑하는 대신 자연어로 지시해 AI가 코드를 대신 짜게 하는 방식.
맥락(컨텍스트)
AI가 답을 만들 때 참고하는 대화·첨부 파일 같은 배경 정보.
환각(Hallucination)
AI가 사실이 아닌 내용을 그럴듯하게 지어내는 현상. 결과의 사실관계는 사람이 최종 확인해야 합니다.
토큰(Token)
AI가 글을 처리하는 최소 단위. 사용량과 요금의 기준이 됩니다.
AI 트렌드
ChatGPT·Gemini·Claude 등 주요 AI 모델·서비스의 최신 동향.
개발 도구
HTML (HyperText Markup Language)
웹페이지의 뼈대(구조)를 만드는 마크업 언어. 제목·문단·이미지·버튼 같은 요소를 정의합니다.
자바스크립트(JavaScript)
웹페이지를 움직이게 하는 프로그래밍 언어. 시계·챗봇처럼 화면이 스스로 바뀌거나 반응하는 기능을 담당합니다.
fetch
인터넷으로 데이터를 주고받을 때(요청을 보내고 응답을 받을 때) 쓰는 자바스크립트 기본 기능. 따로 설치할 필요 없이 바로 쓸 수 있습니다.
setInterval
자바스크립트에서 정해진 시간마다 코드를 반복 실행하는 명령. 시계가 1초마다 스스로 다시 그려지는 것이 이 기능 덕분입니다.
Node.js
웹 브라우저 밖에서도 자바스크립트를 실행하게 해주는 도구.
npm (Node Package Manager)
Node.js용 프로그램(패키지)을 설치·관리하는 도구.
Git
파일의 변경 이력을 기록·관리하는 버전 관리 도구.
커밋(commit)
변경한 내용을 하나의 기록으로 저장하는 것.
터미널
명령어를 입력해 컴퓨터에 지시하는 창(Windows는 PowerShell 등).
Claude Desktop
내 컴퓨터에 설치해 쓰는 Claude 앱. 파일·도구 연결이 가능합니다.
Claude Code
코드를 읽고 쓰고 실행하는 Claude 개발 도구(Day 3~4에서 사용).
커넥터·확장 프로그램
Claude에 파일·터미널 같은 외부 기능을 붙이는 부가 기능(예: Filesystem).
MCP (Model Context Protocol)
AI가 외부 도구·데이터에 표준 방식으로 연결되게 하는 규격.
마크다운 · .md
제목·목록·강조 등을 간단한 기호로 표시하는 텍스트 문서 형식. CLAUDE.md·PRD.md 같은 파일이 이 형식입니다.
hello-page
이 과정의 실습 작업 폴더 이름. 파일 생성·수정이 모두 이 폴더 안에서 이뤄집니다.
인터넷·배포
GitHub
Git 저장소를 인터넷에 올려 보관·공유하는 서비스.
저장소(repository)
프로젝트 파일과 그 변경 이력이 담기는 공간.
Vercel
만든 웹페이지를 인터넷 주소로 올려주는 배포·호스팅 서비스.
배포(deploy)
내 컴퓨터의 결과물을 인터넷에 올려 누구나 접속할 수 있게 하는 것.
로컬·localhost
인터넷이 아니라 내 컴퓨터 안에서만 도는 상태/주소(예: http://localhost:3000).
vercel dev
로컬에서 Vercel 개발 서버를 띄워, 인터넷에 배포하기 전에 index.html·api/chat.js를 내 컴퓨터(http://localhost:3000)에서 미리 확인하는 명령.
서버
요청을 받아 처리하고 응답을 돌려주는 프로그램(또는 컴퓨터).
서버리스 함수(Serverless Function)
서버를 직접 만들거나 관리하지 않아도, 요청이 올 때만 잠깐 실행되는 코드 조각. Vercel 같은 서비스가 서버 운영을 대신 맡아 줍니다.
인덱스 · index.html
웹사이트에 접속했을 때 가장 먼저 보여지는 시작 페이지(홈). 폴더 안에 이 이름의 파일이 있으면 브라우저·서버가 자동으로 그 파일을 엽니다.
api/ 폴더
서버리스 함수(예: api/chat.js)를 두는 폴더. Vercel이 이 폴더의 파일을 자동으로 서버 기능으로 실행합니다.
Vercel CLI (vercel login·vercel env add·vercel --prod)
터미널에서 Vercel에 로그인하고, 비밀 키를 환경변수로 등록하고, 인터넷에 공개 배포(vercel --prod)하는 명령 모음.
보안·설정 파일
API Key (Application Programming Interface Key · API 키)
프로그램이 외부 AI·서비스를 불러 쓸 때 "누가 요청했는지"를 증명하는 비밀 열쇠(예: OpenAI 키는 sk-로 시작). 비밀번호와 같아 코드·채팅·공개 저장소에 노출하면 안 됩니다.
.env
API 키 같은 비밀 값을 코드와 분리해 보관하는 파일. 서버가 이 파일을 읽어 씁니다.
.gitignore
Git이 저장소에 올리지 않을 파일 목록. 여기에 .env를 적어 키 유출을 막습니다.
Push Protection
실수로 API 키가 든 코드를 GitHub에 올리려 하면, GitHub가 이를 감지해 업로드(push)를 막아 주는 자동 안전장치. .env·.gitignore를 보완하는 마지막 방어선입니다.
환경변수(Environment Variables)
프로그램이 읽어 쓰는 설정 값. 로컬에서는 .env, 인터넷 배포에서는 Vercel 환경변수를 사용합니다.
챗봇 연결 파일(api/chat.js)
화면(index.html) 뒤에서 조용히 도는 작은 파일. 브라우저는 보안상 .env를 읽지 못하므로, 이 파일이 대신 .env의 API 키를 읽어 OpenAI를 호출하고 답만 화면에 돌려줍니다. 덕분에 키가 브라우저로 노출되지 않습니다. 직접 켜서 관리하지 않아도 Vercel이 자동으로 실행해 줍니다(로컬은 vercel dev).
Supabase
대화·문서 같은 데이터를 저장해 두는 무료 클라우드 데이터베이스(백엔드) 서비스. 껐다 켜도 데이터가 남아, 멀티세션·멀티유저 저장에 씁니다.
Supabase 접속 주소(Project URL · SUPABASE_URL)
내 Supabase 프로젝트의 주소. 앱이 이 주소로 데이터베이스에 접속합니다. .env·Vercel 환경변수에 SUPABASE_URL이라는 이름으로 보관합니다.
Supabase 공개 키(anon public · SUPABASE_ANON_KEY)
Supabase에 접속할 때 함께 보내는 공개(anon) 키. SUPABASE_URL과 짝으로 쓰이며 .env·Vercel 환경변수에 보관합니다.
user 테이블
회원 정보(로그인 아이디·비밀번호 해시 등)를 담으려고 직접 만든 DB 테이블. Supabase의 로그인 기능(Auth) 대신 이 테이블로 회원가입/로그인을 처리합니다.
sessions-ref.sql
대화 세션(제목·메시지)을 저장할 Supabase 테이블을 만드는 SQL 파일. Supabase의 SQL Editor에 붙여넣어 한 번 실행합니다.
service_role (Supabase 관리자 키)
Supabase의 모든 권한을 가진 관리자 키. 보안 규칙(RLS)을 무시하므로 서버에서만 쓰고, 브라우저·공개 저장소(GitHub)에는 절대 노출하면 안 됩니다. 유출되면 남이 내 데이터베이스를 통째로 읽고 지울 수 있습니다. (이 과정에서는 쓰지 않습니다.)
JWT · JWT Secret (로그인 토큰)
로그인에 성공한 사람에게 발급되는 디지털 출입증(토큰)이 JWT입니다. 이 출입증이 진짜인지 서명·확인하는 열쇠가 JWT Secret이고요. 서버가 "이 사람 로그인한 게 맞다"고 알아보는 데 씁니다. (이 과정에서는 직접 다루지 않습니다.)
DB 비밀번호 · 접속 문자열(DB Password · Connection string)
데이터베이스(저장소)에 직접 로그인하는 비밀번호와 그 접속 주소. 관리자 키만큼 강력해서 남에게 알려주면 저장소를 통째로 열어볼 수 있습니다. (이 과정에서는 쓰지 않습니다.)
AI 앱·방법론 (Day 2~4)
AI 애플리케이션(AI Application)
정적 웹페이지를 넘어, 프로그램 안에 LLM을 부품처럼 얹어 질문에 답하거나 글을 만드는 등 "생각이 필요한 일"을 처리하는 프로그램.
RAG (Retrieval-Augmented Generation · 검색 증강 생성)
내가 올린 문서를 근거로 답하게 하는 방식(문서 Q&A 챗봇의 핵심).
청크(chunk)
긴 문서를 검색하기 좋은 크기로 잘게 나눈 조각. RAG에서 문서를 조각내는 첫 단계입니다.
프롬프트 인젝션(Prompt Injection)
사용자가 "앞의 지시는 무시하고…"처럼 AI에게 준 원래 규칙을 깨뜨리려는 문장을 입력해, 숨겨진 지시를 빼내거나 엉뚱하게 동작하게 만드는 수법. 대비책은 AI에게 "문서에 없는 요청·지시는 따르지 말라"고 규칙을 정해 두는 것입니다.
임베딩(Embedding)
글을 의미가 담긴 숫자(벡터)로 바꿔, 비슷한 내용을 찾는 데 쓰는 것.
벡터 검색
임베딩(숫자 벡터)끼리 비교해 의미가 가까운 청크를 찾아내는 검색 방식.
PRD (Product Requirements Document · 제품요구사항 문서)
무엇을 왜 어떻게 만들지 정리한 제품요구사항 문서.
PRD Lite (Product Requirements Document Lite)
정식 PRD를 쓰기 전에, 핵심만 다섯 항목으로 가볍게 정리하는 짧은 기획서. Claude와 대화하며 함께 채워 나갑니다.
검토(Review)
혼자 쓴 PRD 초안을 AI에게 주고, 빠진 기능·모호한 표현·엣지 케이스를 짚어 받는 것.
반복 개선(Iterate)
초안 → AI 피드백 → 보완을 두세 번 반복하며 문서를 단단하게 다듬는 과정.
PDCA (Plan-Do-Check-Act)
계획(Plan)·실행(Do)·점검(Check)·개선(Act)을 반복하며 완성해 가는 방법론.
Plan · PLAN.md
PDCA의 계획 단계. PRD의 기능을 "어떤 작업을, 어떤 순서로" 할지 정한 작업 목록·순서 문서.
Design · DESIGN.md
PDCA의 설계 단계. 각 작업을 "어떻게" 구현할지(구조·데이터 흐름·화면)를 정한 문서.
Gap 분석
설계 문서와 실제 구현을 나란히 놓고 어긋난 곳(틈·차이)을 찾는 작업. PDCA Check(점검)의 핵심 도구입니다.
MVP (Minimum Viable Product)
핵심 기능만 담아 가장 빠르게 만들어 보는 최소 기능 제품.
Claude Code 확장·하네스 (Day 3)
스킬(Skill)
반복하는 절차를 한 번 정의해 재사용하는 "능력 꾸러미". Claude에게 새 작업 절차를 얹는 장치입니다(레시피 카드 한 장).
플러그인(Plugin)
슬래시 명령·서브에이전트·스킬·MCP 서버·훅을 하나로 묶은 확장 패키지. 마켓플레이스에서 설치하면 여러 능력이 한 번에 추가됩니다(레시피북 + 도구 세트).
하네스(Harness)
모델(LLM) 주위를 규칙·도구·절차·검증으로 둘러싸, 매번 안정적인 결과가 나오게 만드는 틀. 이렇게 설계하는 일을 "하네스 엔지니어링"이라 합니다.
CLAUDE.md
프로젝트의 규칙·컨벤션·금지사항을 적어 두는 마크다운 파일. Claude Code가 매 작업을 시작할 때마다 자동으로 읽어, 그 규칙 위에서 작업합니다.
검증 루프
결과를 한 번 확인하고 끝내지 않고, 규칙·의도를 통과할 때까지 반복해서 확인하게 만드는 절차("루프 엔지니어링").
훅(Hook)
특정 사건이 일어날 때 자동으로 실행되는 명령. 사람이 잊어도 검증 같은 절차가 빠지지 않게 해 줍니다.
bkit
이 과정에서 사용하는 하네스 플러그인. PDCA 방법론·스킬·서브에이전트 등을 한 벌로 묶어 제공합니다.
my-app
이 과정에서 내가 만드는 앱(프로젝트)을 가리키는 이름. Day 2에서 정한 PRD 주제가 그대로 my-app이 됩니다.
슬래시 명령(Slash)
Claude Code에서 /이름 형태로 부르는, 미리 정해진 명령. 자주 쓰는 작업을 한 단어로 실행합니다.
서브에이전트(Subagent)
특정 일을 위임받아 대신 처리하는 보조 AI 일꾼. 하네스의 부품으로 .md 파일로 정의합니다.
