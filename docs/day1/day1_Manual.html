1. AI 기초개념과 트렌드

Generative AI — 생성형 AI란 무엇인가

생성형 AI(Generative AI)는 기존 데이터를 분류·예측하는 데 그치지 않고, 글·이미지·코드처럼 새로운 결과물을 직접 만들어내는 인공지능을 말합니다. 이번 과정에서 활용할 Claude 역시 대표적인 생성형 AI로, 우리가 한국어로 지시하면 그에 맞는 코드와 문서를 새로 만들어 줍니다.

LLM — "정답을 아는 존재"가 아니라 "그럴듯한 다음 단어를 예측하는 존재"

Claude·GPT와 같은 대규모 언어 모델(LLM, Large Language Model)은 방대한 글을 학습해, 주어진 문맥 다음에 올 확률이 가장 높은 단어를 순서대로 이어 붙이는 방식으로 문장을 만듭니다. 즉 "정답이 어딘가 저장돼 있어 꺼내 오는 것"이 아니라, "지금까지의 맥락을 보고 가장 그럴듯한 다음 문장을 매번 새로 생성하는 것"에 가깝습니다. 이 원리를 이해하면 왜 같은 질문에도 답이 조금씩 달라지는지, 왜 맥락을 충분히 줄수록 결과물이 좋아지는지 자연스럽게 이해할 수 있습니다.

context — 맥락을 주는 만큼 결과물이 좋아진다

LLM은 대화창에 입력된 내용, 첨부 파일, 이전 대화 기록 등 "맥락"을 근거로 답을 만듭니다. "요약해줘"보다 "핵심 쟁점과 숫자 위주로 6줄 이내로 요약해줘"처럼 맥락을 구체적으로 줄수록 실무에 바로 쓸 수 있는 결과가 나옵니다. 이 원리는 뒤의 바이브 코딩에서 프롬프트를 잘 쓰는 법으로 이어집니다.

hallucination — AI가 잘하는 일과 사람이 판단할 일

LLM은 반복적인 코드 작성, 문서 초안, 오류 메시지 해석, 절차 설명에 매우 능숙합니다. 반면 존재하지 않는 사실을 그럴듯하게 지어내는 "환각(hallucination)"이 나타날 수 있으므로, 사실관계의 최종 검증과 정책적 판단은 반드시 담당자가 직접 확인해야 합니다.

AI Trend — 최신 모델 동향 (ChatGPT · Gemini · Claude)

현재 생성형 AI 시장은 세 개의 해외 프런티어 모델이 이끌고 있습니다. OpenAI의 ChatGPT, Google의 Gemini, Anthropic의 Claude입니다. 셋 다 글·이미지·코드를 다루는 범용 대화형 AI라는 점은 같지만, 강점과 잘 어울리는 업무가 조금씩 다릅니다.
세부 성능·가격은 각 제공사 공식 페이지에서 계속 갱신되므로, 실제 도입 시에는 최신 정보를 확인하시기 바랍니다.

모델 (제공사)대표 강점특징 · 잘 맞는 업무
ChatGPT
(OpenAI)		범용성 · 생태계가장 널리 쓰이는 대화형 AI로, 폭넓은 API·플러그인 생태계와 이미지 생성·음성 대화 등 다양한 기능을 한곳에서 제공합니다. 일반적인 문서 작성·아이디어 발상에 두루 강합니다.
Gemini
(Google)		구글 연동 · 멀티모달구글 검색·Gmail·문서 등 구글 서비스와의 연동과 최신 정보 반영에 강하며, 텍스트·이미지·영상을 함께 이해하는 멀티모달 처리와 매우 긴 문맥 처리에 강점이 있습니다.
Claude
(Anthropic)		긴 문서 · 코드 · 안전성긴 문맥의 정확한 이해, 코드 작성, 안정적이고 신중한 답변에 강점이 있습니다.
이번 과정의 주력 도구로, 터미널에서 직접 파일을 다루는 Claude Code의 기반이 됩니다.

🇰🇷 참고 — 국산 모델: 국내에서도 Upstage의 Solar, 네이버의 HyperCLOVA X 등 한국어에 최적화된 LLM이 함께 발전하고 있습니다. 데이터 주권·한국어 성능이 중요한 공공 업무에서는 국산 모델도 함께 검토할 수 있습니다.

💡 핵심 한 줄: LLM은 맥락을 주면 그럴듯한 다음 문장을 만들어내는 도구이지, 정답을 아는 존재가 아닙니다 — 최종 판단은 언제나 사람의 몫입니다.

2. 온라인 서비스의 이해 — GitHub · Vercel · OpenAI

GitHub — 코드를 보관하고 공유하는 원격 저장소

Git이 내 컴퓨터 안에서 이력을 관리한다면, GitHub는 그 코드를 인터넷 저장소(repository)에 올려 보관·공유·협업할 수 있게 해주는 서비스입니다. 뒤에서 사용할 Vercel이 GitHub 계정과 연동되기 때문에 함께 준비합니다.

Vercel — 만든 웹페이지를 인터넷에 공개하는 배포 서비스

내 컴퓨터에만 있는 웹페이지는 나만 볼 수 있습니다. Vercel은 그 파일을 자사 서버에 올려 누구나 접속할 수 있는 인터넷 주소(URL)로 만들어 주는 배포·호스팅 서비스입니다. "로컬(내 컴퓨터)"과 "배포(인터넷 공개)"의 차이는 5장에서 더 다룹니다.

API Key — 프로그램이 AI를 불러 쓰기 위한 비밀 열쇠

사람이 화면에서 로그인하듯, 프로그램이 외부 AI 서비스를 호출할 때는 "누가 요청했는지"를 증명해야 합니다. 그 증명 수단이 API Key(sk-로 시작하는 긴 문자열)입니다. 이후 실습에서 OpenAI 기능을 프로그램으로 불러 쓸 때 필요하므로 미리 발급해 둡니다.

API Key는 비밀번호와 같습니다. 코드·채팅·공개 저장소에 노출하면 타인이 내 계정으로 요금을 발생시킬 수 있습니다.
발급 화면을 벗어나면 다시 볼 수 없으므로 안전하게 보관하고, 유출이 의심되면 즉시 삭제(Revoke) 후 재발급해야 합니다.
서비스비유역할
GitHub코드 보관 창고코드를 인터넷에 저장·공유
Vercel공개 게시대웹페이지를 인터넷 주소로 배포
API Key출입 열쇠프로그램이 AI를 호출할 때 신원 증명
세 계정을 실제로 가입하고 API Key를 발급하는 과정은 실습 STEP 1에서 진행합니다.

3. 개발 도구의 이해 — Node.js와 Git

Node.js — 개발 도구를 돌리는 실행 환경

Node.js는 원래 웹 브라우저 안에서만 동작하던 자바스크립트를, 컴퓨터에서 직접 실행할 수 있게 해주는 프로그램입니다. 이번 과정에서 사용하는 여러 도구(Claude Code, Vercel 명령어 등)는 Node.js와 함께 설치되는 npm이라는 "도구 설치기"를 통해 내려받습니다. 그래서 Node.js가 가장 먼저 준비되어야 합니다.

npm(Node Package Manager)은 앱스토어처럼 개발 도구를 이름만으로 설치·관리해 주는 도구입니다. 실습에서 npm install 명령이 자주 나오는 이유가 바로 이것입니다.

Git — 코드의 변경 이력을 지키는 안전벨트

Git은 파일이 언제·무엇이 바뀌었는지 이력을 기록하고, 필요하면 과거로 되돌릴 수 있게 해주는 버전 관리 도구입니다. 하나의 저장 지점을 "커밋(commit)"이라 하며, 이 커밋 이력이 쌓여 프로젝트의 역사가 됩니다. 뒤에서 다룰 GitHub·Vercel 배포는 모두 이 Git을 바탕으로 동작합니다.

도구한 줄 정의이번 과정에서의 쓰임
Node.js자바스크립트 실행 환경Claude Code·Vercel 등 도구 설치의 토대
npm도구 설치·관리기npm install로 도구를 내려받음
Git코드 변경 이력 관리기GitHub 업로드·Vercel 배포의 바탕
이 개념들을 실제로 설치하고 node -v·git --version으로 확인하는 과정은 실습 STEP 2에서 진행합니다.

4. Claude와 Claude Code의 이해

Free / Pro — LLM 구독이란

Claude는 무료(Free)로도 채팅을 쓸 수 있지만, 이번 과정의 주력 도구인 Claude Code는 유료 구독(Pro 이상)에서만 동작합니다. 구독은 "더 많은 사용량과 추가 기능을 쓰기 위한 월 단위 이용권"이라고 이해하면 됩니다.

구분Free 플랜Pro 플랜
가격무료월 $20 (연간 결제 시 월 $17)
이용 범위웹·앱 채팅채팅 + Claude Code 이용 가능
사용량기본Free 대비 대폭 확대, 혼잡 시 우선 접속

Claude Code — 대화만 하는 AI에서, 파일을 직접 다루는 AI로

claude.ai 채팅이나 Claude Desktop에서는 Claude와 "대화"만 할 수 있습니다. 반면 Claude Code는 터미널(명령어 창) 안에서 Claude에게 한국어로 지시하면, Claude가 실제로 내 컴퓨터의 파일을 만들고 수정해 줍니다. 즉 "말로 설명해 주는 AI"에서 "직접 작업해 주는 AI"로 한 걸음 나아간 도구입니다.

터미널 · 명령어 — CLI라는 대화 방식

터미널(CLI, Command Line Interface)은 버튼 대신 글자 명령으로 컴퓨터에 지시하는 화면입니다. Claude Code를 실행하면 여기에 한국어 문장을 쓰거나, /로 시작하는 "슬래시 명령어"로 도구 자체를 제어합니다. 아래는 각 기본 명령어가 하는 일의 의미입니다(실제 입력은 실습에서).

명령어개념적 역할
claude현재 폴더에서 Claude Code와의 대화를 시작
/help쓸 수 있는 명령어 목록을 보여줌
/clear지금까지의 대화 맥락을 초기화(새 주제 시작 시)
/model사용할 Claude 모델을 바꿈
/login계정 로그인(최초 1회 인증)

CLI vs IDE — Claude Code를 쓰는 두 가지 환경

Claude Code는 두 가지 방식으로 쓸 수 있습니다. 하나는 CLI(터미널)에서 claude 명령으로 실행하는 방식이고, 다른 하나는 Cursor 같은 IDE(코드 편집기)에 확장(익스텐션)으로 설치해 화면 옆 패널에서 쓰는 방식입니다. Cursor는 AI 코딩에 특화된 편집기로, 코드 파일·미리보기·터미널을 한 화면에서 다룰 수 있어 초보자에게 편합니다. 이번 과정의 실습은 Cursor(IDE) 환경에서 진행합니다.

Claude 구독 Claude Code CLI · IDE(Cursor)

Pro 구독과 CLI · IDE(Cursor) 두 환경 준비, 기본 명령어 실행은 실습 STEP 3에서 진행합니다.

5. 바이브 코딩과 배포의 이해

Vibe Coding — 자연어로 소프트웨어를 만든다는 것

바이브 코딩(Vibe Coding)은 프로그래밍 문법을 직접 타이핑하는 대신, "이런 걸 만들어 줘"라고 자연어로 지시해 AI가 코드를 대신 작성하게 하는 방식입니다. 코드를 한 줄도 몰라도 결과물을 만들 수 있다는 것이 핵심이며, 이번 실습의 헬로페이지가 그 첫 경험이 됩니다.

Prompt — 좋은 결과를 부르는 지시문의 원리

같은 도구라도 어떻게 지시하느냐에 따라 결과가 크게 달라집니다. 1장의 맥락(context) 원리가 여기에 그대로 적용됩니다.
좋은 프롬프트는 대체로 아래 세 요소를 담습니다.

요소의미예
역할어떤 입장에서 작업할지 지정"너는 웹 디자이너야"
구체성무엇을·어떤 형태로 원하는지 명시"제목, 소개 문단, 목록 3개를 넣어"
제약스타일·범위·조건을 한정"파란색 계열로 깔끔하게"

deploy — 로컬과 배포의 차이

내 컴퓨터 안의 파일을 브라우저로 여는 것은 "로컬(local)" 상태로, 나만 볼 수 있습니다. 이 파일을 Vercel 같은 서비스로 올려 인터넷 주소를 부여하는 것을 "배포(deploy)"라 하며, 그 순간부터 그 주소를 아는 누구나 접속할 수 있습니다. 실습의 마지막 단계에서 헬로페이지를 실제로 배포해, 로컬 → 인터넷 공개의 전 과정을 경험합니다.

💡 핵심 한 줄: 자연어로 만들고(바이브 코딩), 명령어 한 줄로 세상에 공개하는(배포) 흐름 — 이 개념을 손으로 확인하는 것이 Day 1 실습의 목표입니다.
