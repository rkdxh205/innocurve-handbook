STEP 1 계정 GitHub · Vercel · OpenAI 계정 3종 준비하기
아직 아무것도 없는 첫 시작입니다. 이후 실습에서 쓸 온라인 계정 세 가지를 먼저 만들어 둡니다.
GitHub(코드 저장), Vercel(배포), OpenAI(API Key)를 순서대로 준비하면 뒤 실습에서 멈추지 않습니다.
관련 이론
GitHub·Vercel·OpenAI의 이해
단계별 실행
  1. GitHub 가입

    github.com에 접속해 우측 상단 "Sign up"을 클릭한 뒤 순서대로 입력합니다.(구글 계정이나 애플 계정으로도 가입할 수 있습니다 — 이 경우 "Continue with Google" 또는
    "Continue with Apple"을 선택하면 이메일·비밀번호 입력 없이 해당 계정으로 바로 가입됩니다.)

    1. 이메일을 입력하고 Continue를 누릅니다.
    2. 비밀번호를 입력하고 Continue를 누릅니다.
    3. 사용자 이름(username, 영문·숫자)을 입력하고 Continue를 누릅니다.
    4. 사람 확인(퍼즐)을 풀고, 입력한 이메일로 온 인증 코드를 입력하면 가입이 완료됩니다.
  2. Vercel 가입 (GitHub 연동)

    vercel.com에 접속해 우측 상단 "Sign Up"을 클릭합니다.

    1. 계정 유형에서 개인 실습용 "Hobby"를 선택하고 이름을 입력한 뒤 Continue합니다.
    2. "Continue with GitHub" 버튼을 클릭합니다.
    3. GitHub 로그인 화면이 뜨면 로그인하고, "Authorize Vercel"을 클릭해 연동을 허용합니다.
    4. Vercel 대시보드가 열리면 가입이 완료된 것입니다.
  3. OpenAI 결제 등록 및 API Key 발급

    platform.openai.com에 로그인(또는 가입)합니다. OpenAI API는 사용한 만큼 선불 크레딧에서 차감되는 방식이라,
    키를 발급하기 전에 먼저 결제 수단을 등록하고, 최소 금액을 충전해야 키가 정상 동작합니다.

    1. 좌측 하단(또는 우측 상단) 톱니바퀴 Settings → "Billing"으로 이동합니다.
    2. "Add payment details"(결제 수단 추가)를 클릭하고, 개인/법인 카드 정보를 입력해 등록합니다.
    3. 충전할 금액을 입력합니다 — 실습용으로는 최소 금액인 $5면 충분합니다.
    4. 자동 충전(정기 결제, "Enable auto recharge") 옵션은 반드시 체크 해제합니다.(잔액이 떨어질 때 카드에서 자동으로 다시 결제되는 것을 막기 위함입니다.)
    5. 결제를 완료하고 크레딧(잔액)이 충전되었는지 확인합니다.
    6. 좌측 메뉴에서 "API keys"로 이동해 "Create new secret key" 버튼을 클릭합니다.
    7. 키 이름을 입력하고 "Create secret key"를 클릭합니다.
    8. 표시된 sk-로 시작하는 키의 "Copy"를 눌러 복사한 뒤 안전한 곳에 보관하고 "Done"을 누릅니다.(이 화면을 벗어나면 키를 다시 볼 수 없습니다.)
API Key는 비밀번호와 같습니다. 코드·채팅·공개 저장소에 노출하지 마십시오. 발급 화면을 벗어나면 다시 볼 수 없으므로 반드시 복사해 두고, 유출이 의심되면 즉시 삭제(Revoke) 후 재발급하십시오. 또한 예기치 않은 과금을 막기 위해 자동 충전(auto recharge)은 꺼 두고, 충전한 $5 한도 안에서만 사용하십시오.
● 작업 도구 Toolchain AI 작업대(Claude Desktop)와 개발 도구(Node.js·Git)를 갖춥니다
STEP 2 설치 Claude 구독 + Claude Desktop 사용 환경 준비 (로컬 작업 폴더 지정)
이번 과정의 작업 도구인 Claude Desktop을 쓸 수 있게 만듭니다.
Claude를 Pro로 구독하고 앱을 설치한 뒤, 로컬 작업 폴더로 hello-page를 지정해 Claude가 그 폴더에 직접 파일을 만들 수 있게 합니다.(앞으로의 모든 실습은 이 Claude Desktop 환경에서 진행합니다.)
관련 이론
Claude·Claude Desktop의 이해
단계별 실행
  1. Claude 가입 및 유료 구독 (Pro 또는 Max)

    claude.ai에서 계정을 만든 뒤 아래 순서로 유료 플랜을 구독합니다.

    1. 로그인 후 화면 좌측 하단의 이름을 클릭하고 "Settings"를 선택합니다.
    2. "Billing" 탭으로 이동해 "Upgrade plan"을 클릭합니다.
    3. 실습 강도에 맞는 플랜을 선택하고 결제 주기를 고릅니다.(아래는 2026년 7월 기준 요금이며, 최신 금액은 결제 화면에서 확인하세요.)
      • Pro — 월 $20(연간 결제 시 약 $17/월). 개념 실습·PRD 작성 등 가벼운 작업에 적당합니다.
      • Max 5× — 월 $100. Pro의 약 5배 사용량. 본격 앱 제작 단계에 여유가 있습니다.
      • Max 20× — 월 $200. Pro의 약 20배 사용량. 장시간 집중 제작에 적합합니다.(Max는 월간 결제만 제공되며 연간 할인이 없습니다.)
    4. 카드 정보를 입력하고 "Subscribe"를 눌러 구독을 완료합니다.
    뒤 과정의 Claude Code는 파일을 읽고·쓰고·검증하는 작업을 반복해 토큰이 빠르게 소모됩니다. 고도화된 PRD 작성이나 앱 제작 같은 본격 단계에서는 Pro만으로는 사용량 한도에 금방 도달할 수 있습니다. 그래서 집중 제작 주간에는 Max를 권장하고, 이후 사용 빈도를 보며 Pro로 낮추는 방식을 추천합니다. 비용이 부담되면 Pro로 시작해도 되지만, 실습 중 한도에 걸리면 그날 진도가 끊길 수 있다는 점만 감안하세요.
  2. Claude Desktop 설치 및 로그인

    내 컴퓨터에 Claude 데스크톱 앱을 설치합니다.

    1. 브라우저에서 claude.ai/download 에 접속합니다.
    2. 운영체제에 맞는 설치 파일을 내려받습니다.
      • Windows: .exe 설치 파일
      • macOS: .dmg 파일 (Apple 실리콘·인텔 자동 감지)
    3. 내려받은 파일을 실행해 안내에 따라 설치를 완료합니다.
    4. Claude Desktop을 실행하고, 앞에서 유료 플랜을 구독한 계정으로 로그인합니다.
  3. 실습용 작업 폴더 만들기

    Claude가 만든 파일이 담길 전용 폴더를 하나 준비합니다. 나중에 이 폴더 하나만 Claude에게 열어 줄 거라서, 다른 파일과 섞이지 않도록 새 폴더로 따로 만드는 것이 좋습니다.

    1. 바탕화면처럼 나중에 다시 열기 쉬운 곳으로 갑니다. 바탕화면이 가장 편하고, 문서 폴더나 내 사용자 폴더도 괜찮습니다 — 위치는 어디든 상관없습니다.
    2. 빈 공간에서 마우스 오른쪽 버튼 → "새 폴더"를 눌러 폴더를 만들고, 이름을 hello-page로 바꿉니다.(폴더 이름은 한글·띄어쓰기 없이 영문으로 지어 주세요. 그래야 나중에 오류가 나지 않습니다.)

    윈도우에서 바탕화면이 OneDrive로 백업되고 있어도 그대로 만드시면 됩니다. 혹시 나중에 폴더를 못 찾거나 경로 관련 오류가 나면, 내 사용자 폴더(C:\Users\내이름)에 만들면 가장 안전합니다.

  4. Claude Desktop에 로컬 작업 폴더 연결하기

    Claude가 방금 만든 hello-page 폴더에서 파일을 직접 만들고 고칠 수 있도록, "Filesystem(파일시스템)"이라는 기능을 켜고 이 폴더를 열어 줍니다. 아래 순서대로 클릭만 하면 됩니다.

    1. 앱 설정을 엽니다. 창 왼쪽 맨 아래의 내 이름(프로필)을 클릭한 뒤 "설정"을 누릅니다.(키보드로 Ctrl + ,(쉼표)를 눌러도 열립니다. 채팅창 안의 톱니바퀴가 아니라 앱 전체 설정이어야 합니다.)
    2. 설정 창 왼쪽 메뉴에서 "데스크톱 앱" 아래의 "확장 프로그램"을 클릭합니다.(같은 왼쪽 메뉴 아래쪽에 있는 "커넥터"와 헷갈리지 마세요. 폴더 연결은 "확장 프로그램"에서 합니다.)
    3. 오른쪽 위 "확장 프로그램 찾아보기"를 눌러, 목록에서 "Filesystem"을 찾아 클릭합니다.("Let Claude access your filesystem to read and write files."라고 설명된 항목입니다.)
    4. Filesystem 화면에서 설치한 뒤 구성 화면으로 들어갑니다.("구성" 버튼이 보이면 클릭하세요. "활성화하려면 먼저 필수 항목을 채우라"는 안내가 있는 화면입니다.)
    5. 구성 화면의 "Allowed Directories(허용 디렉터리)"에서 "+ directory 추가" 버튼을 눌러, 앞서 만든 hello-page 폴더만 선택합니다.(바탕화면 전체나 여러 폴더가 아니라 딱 이 실습 폴더 하나만 추가해야 안전합니다.)
    6. "저장"을 누른 뒤 맨 위 토글을 켜 "사용 중" 상태로 만듭니다. 토글이 파란색으로 바뀌고 빨간 경고가 사라지면 연결 완료입니다.(추가한 폴더 경로가 C:\Users\사용자명\Desktop\hello-page처럼 표시되면 제대로 지정된 것입니다. 폴더를 추가하기 전에는 저장 버튼이 회색으로 눌리지 않습니다.)
    7. 제대로 됐는지 확인: 대화창 아래 입력칸 왼쪽의 + 버튼을 눌러 "커넥터" 목록에 Filesystem이 보이면 연결된 것입니다.

    구성 화면을 아래로 내리면 "도구 권한"이 있는데, 기본값("승인 필요") 그대로 두면 됩니다 — Claude가 파일을 읽거나 만들기 전에 매번 허용을 물어보므로 더 안전합니다.

    연결 확인: 대화창에 "hello-page 폴더에 test.txt 파일을 직접 만들어줘"라고 입력하세요. 도구 사용 허용 팝업이 뜨면 "허용"을 누르고, 실제 hello-page 폴더에 test.txt가 생기면 성공입니다. (확인 후 지워도 됩니다.)
    꼭 기억하세요 — "직접, 실제 폴더에": Claude에게 파일을 만들거나 고쳐 달라고 할 때 "직접"·"실제 폴더에" 같은 말을 넣으면 실제 폴더에 확실히 반영됩니다. 이 말이 없으면 Claude가 폴더 대신 "다운로드 및 열기" 버튼이 달린 임시 파일만 만들어 줄 수 있습니다. 그런 파일만 보이고 폴더엔 안 생긴다면 "다운로드 파일 말고 실제 폴더에 직접 써줘"라고 다시 요청하세요. 이 규칙은 앞으로 모든 실습(웹페이지·PRD·앱 제작)에 똑같이 적용됩니다.
작업 폴더로는 실습 폴더(hello-page)만 지정하십시오. 내 문서·바탕화면 전체나 시스템 폴더를 지정하면 Claude가 그 폴더의 파일도 다룰 수 있게 되어 위험합니다.
앞으로의 모든 실습(STEP 4부터)은 이 Claude Desktop 환경에서 진행합니다. 즉 Claude Desktop 대화창에 한국어로 지시하면, 지정한 hello-page 폴더에 파일이 직접 만들어지는 방식입니다.
STEP 3 설치 Desktop Commander 연결하고, Node.js·Git은 Claude에게 맡기기
작업대(Claude Desktop)를 갖췄으니, 모든 실습의 토대인 Node.js와 Git을 준비합니다.
Claude Desktop에 "Desktop Commander"라는 커넥터를 클릭 한 번으로 연결하면, 이후로는 터미널을 전혀 열지 않고 Node.js 설치부터 Git 설치·버전 확인까지 채팅으로 끝낼 수 있습니다.
관련 이론
Node.js·Git의 이해
단계별 실행
1️⃣ Desktop Commander 연결 — 클릭만으로, 터미널 없음
  1. 앱 설정 열기

    STEP 2에서 했던 것처럼 창 왼쪽 맨 아래의 내 이름(프로필)을 클릭한 뒤 "설정"을 누릅니다.

  2. 확장 프로그램 디렉터리 열기

    설정 창 왼쪽 메뉴에서 "데스크톱 앱" 아래의 "확장 프로그램"을 클릭한 뒤, 오른쪽 위 "확장 프로그램 찾아보기"를 클릭합니다. "디렉터리" 화면이 열리면 왼쪽에서 "커넥터" 탭을 선택합니다.

  3. Desktop Commander 검색 후 추가

    위쪽 검색창에 "Desktop Commander"를 입력합니다. "Build, explore, and automate on your local machine with access to files and terminal."이라고 설명된 카드가 나오면, 카드 오른쪽의 "+" 버튼을 클릭해 설치합니다.

    1. 설치 진행 중 권한 관련 안내가 나오면 그대로 승인·진행합니다.
    2. 설치가 끝나면 Claude Desktop을 완전히 종료했다가 다시 실행합니다.
  4. 연결 확인

    대화창 아래 입력칸 왼쪽의 + 버튼을 눌러 "커넥터" 목록에 Desktop Commander가 보이면 연결된 것입니다.

2️⃣ Node.js · Git 설치는 채팅으로 — 말로만, 터미널 없음
  1. Claude Desktop 채팅으로 설치 요청

    이제 대화창에 아래처럼 요청하면 Claude가 Desktop Commander를 통해 실제로 설치 명령을 대신 실행합니다.

    프롬프트 
    Node.js LTS 버전이랑 Git이 설치되어 있는지 확인해줘.
없으면 Desktop Commander로 직접 설치해줘.
(Windows면 winget install OpenJS.NodeJS.LTS,
winget install Git.Git /
macOS면 brew install node git)

    Claude가 명령을 실행해도 되는지 물으면 "허용(Allow)"을 누릅니다.

  2. 설치 확인도 채팅으로

    이어서 아래처럼 요청하면 Claude가 직접 버전을 확인해 채팅창에 결과를 보여줍니다.

    프롬프트 
    node -v, npm -v, git --version을
각각 실행해서 결과를 알려줘.

    ▲ 예: v20.x.x, 10.x.x, git version 2.x.x

● 배포 Deploy 웹 결과물을 인터넷에 공개합니다
STEP 8 실습 결과물을 Vercel로 배포하기
내 컴퓨터에만 있던 웹 결과물(자기소개·시계·챗봇·RAG가 모두 담긴 index.html 한 장)을 누구나 접속할 수 있는 인터넷 주소로 배포합니다. "만들고 → 고치고 → 세상에 공개"의 마지막 단계입니다.(챗봇·RAG도 이제 같은 index.html 안의 섹션이라, 파일 하나로 한 번에 배포됩니다.)
관련 이론
배포(deploy)의 개념 Vercel의 이해

완성된 웹 페이지·서버 함수(index.html·api/ 폴더)를 인터넷에 배포합니다. 배포 방법은 두 가지가 있으니, 둘 중 편한 방법 하나를 선택해 진행하면 됩니다.

🌐 방법 ① · GitHub 업로드 후 Vercel 사이트에서 배포 — 이후 자동 재배포
  1. GitHub에 새 저장소(repository) 만들기

    STEP 1에서 만든 GitHub 계정으로 로그인한 뒤, 코드를 올려 둘 빈 저장소를 하나 만듭니다.

    1. github.com 우측 상단의 "+" 아이콘을 클릭하고 "New repository"를 선택합니다.
    2. Repository name에 hello-page를 입력합니다.
    3. 공개 범위는 "Public"을 선택합니다.(README 등 다른 항목은 체크하지 않고 비워 둡니다.)
    4. "Create repository"를 클릭합니다. 다음 화면에 나오는 저장소 주소(https://github.com/사용자명/hello-page.git)를 복사해 둡니다.
  2. 결과물을 GitHub에 업로드하기

    터미널 없이 GitHub 사이트에서 파일을 바로 올릴 수 있습니다. 방금 만든 빈 저장소 화면에서 진행합니다.

    1. 저장소 화면의 "Add file" → "Upload files"를 클릭합니다.
    2. hello-page 폴더에서 index.html·재정경제부.svg·api 폴더(그 안의 chat.js 포함)를 끌어다 놓거나 "choose your files"로 선택합니다.(⚠️ .env는 절대 올리지 마세요 — API Key가 그대로 노출됩니다. server.js는 로컬 테스트 전용이라 올려도 안 올려도 상관없습니다.)
    3. 아래 "Commit changes" 버튼을 눌러 업로드를 확정합니다.
    4. 저장소에 파일들이 올라온 것을 확인합니다.
  3. Vercel 사이트에서 저장소 연결·배포

    vercel.com 대시보드에서 GitHub 저장소를 그대로 불러와 배포합니다.

    1. vercel.com에 로그인하고 "Add New…" → "Project"를 클릭합니다.
    2. GitHub 저장소 목록에서 hello-page를 찾아 "Import"를 클릭합니다.(목록에 없으면 "Adjust GitHub App Permissions"로 저장소 접근을 허용합니다.)
    3. 설정은 기본값 그대로 두고 "Deploy"를 클릭합니다.
    4. 잠시 기다리면 배포가 완료되고 축하 화면과 함께 공개 주소가 나타납니다.
  4. 공개 주소 접속

    Vercel이 만들어 준 https://hello-page-xxxx.vercel.app 주소로 접속해, 내가 만든 페이지가 인터넷에 공개된 것을 확인합니다. 이 주소는 다른 사람에게 공유해도 그대로 접속됩니다.

    이 방식은 GitHub 저장소와 연결되어 있어, 앞으로 코드를 수정해 GitHub에 다시 올리기만 하면 Vercel이 자동으로 다시 배포합니다.
💻 방법 ② · 터미널(Vercel CLI)로 바로 배포 — 명령 한 줄로 빠르게
  1. Vercel CLI 설치 및 로그인

    터미널(Windows는 PowerShell, macOS는 터미널)을 열고, 아래 명령으로 Vercel 도구를 설치한 뒤 STEP 1에서 만든 계정으로 로그인합니다.

    npm install -g vercel
vercel login
  2. 배포 실행

    같은 터미널에서 hello-page 폴더로 이동한 뒤 아래를 실행합니다. 안내 질문에는 대부분 Enter(기본값)로 진행합니다.(폴더 이동: cd hello-page. 이미 그 폴더에 있으면 생략합니다.)

    # 미리 보기 배포
vercel

# 최종(정식) 배포
vercel --prod
  3. 공개 주소 접속

    터미널에 출력된 https://hello-page-xxxx.vercel.app 주소를 브라우저에 입력해, 내가 만든 페이지가 인터넷에 공개된 것을 확인합니다. 이 주소는 다른 사람에게 공유해도 그대로 접속됩니다.

🔑 챗봇·RAG를 위한 마지막 단계 · Vercel에 API 키 등록 — 방법 ①·② 공통, 필수
  1. 왜 필요한가

    지금까지는 server.js가 내 컴퓨터의 .env 파일을 읽어서 index.html의 챗봇·RAG 섹션에 답해 왔습니다. 하지만 .env는 방금 GitHub에 올리지 않았으므로, Vercel 서버(api/chat.js)에는 API 키가 없는 상태입니다. 이 상태에서는 배포된 챗봇이 "API 키가 없습니다" 오류를 보여줍니다.

  2. 환경 변수 등록

    vercel.com에서 방금 배포한 hello-page 프로젝트로 들어갑니다.

    1. 프로젝트 화면 상단의 "Settings" 탭을 클릭합니다.
    2. 왼쪽 메뉴에서 "Environment Variables"를 클릭합니다.
    3. Key에는 OPENAI_API_KEY, Value에는 STEP 1에서 발급한 sk-로 시작하는 내 키를 붙여넣습니다.
    4. 적용 범위는 기본값(Production·Preview·Development 전체 체크)을 그대로 두고 "Save"를 클릭합니다.
  3. 다시 배포(Redeploy)

    환경 변수는 다음 배포부터 적용됩니다. 프로젝트 화면 상단의 "Deployments" 탭 → 가장 최근 배포 옆 "···" 메뉴 → "Redeploy"를 클릭해 다시 배포합니다.

  4. 배포된 챗봇 확인

    재배포가 끝나면 https://hello-page-xxxx.vercel.app에 접속해, 자기소개·시계 아래의 챗봇·RAG 섹션까지 로컬에서 확인했던 것과 똑같이 동작하는지 확인합니다.

Vercel의 Environment Variables에 등록한 키는 서버(함수) 안에서만 쓰이고 브라우저로는 절대 전달되지 않아 안전합니다. 반대로 index.html 같은 화면 코드에 sk-... 키를 직접 적는 것은 절대 금지입니다 — 그건 방금 GitHub에 공개로 올린 파일이라 누구나 볼 수 있습니다.
Vercel은 파일을 자사 서버에 올리고 인터넷에서 접속할 수 있는 고유 주소를 자동으로 만들어 줍니다. 방법 ①은 GitHub 저장소를, 방법 ②는 내 컴퓨터의 폴더를 곧바로 올린다는 점만 다를 뿐, 결과(공개 URL)는 같습니다. api/ 폴더 안의 파일은 Vercel이 자동으로 서버 함수로 인식해 실행합니다.
오늘의 마무리: "자연어로 만들고 → 고치고 → (시계·챗봇·RAG로 확장하고) → 배포하는" 이 흐름이 이후 모든 실습의 기본 골격이 됩니다.
