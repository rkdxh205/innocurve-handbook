STEP 1 계정 GitHub · Vercel · OpenAI 계정 3종 준비하기
아직 아무것도 없는 첫 시작입니다. 이후 실습에서 쓸 온라인 계정 세 가지를 먼저 만들어 둡니다.
GitHub(코드 저장), Vercel(배포), OpenAI(API Key)를 순서대로 준비하면 뒤 실습에서 멈추지 않습니다.
관련 이론
GitHub·Vercel·OpenAI의 이해
단계별 실행
  1. GitHub 가입

    github.com에 접속해 우측 상단 "Sign up"을 클릭한 뒤 순서대로 입력합니다.(구글 계정이나 애플 계정으로도 가입할 수 있습니다 — 이 경우 "Continue with Google" 또는
    "Continue with Apple"을 선택하면 이메일·비밀번호 입력 없이 해당 계정으로 바로 가입됩니다.)

    1. 이메일을 입력하고 Continue를 누릅니다.
    2. 비밀번호를 입력하고 Continue를 누릅니다.
    3. 사용자 이름(username, 영문·숫자)을 입력하고 Continue를 누릅니다.
    4. 사람 확인(퍼즐)을 풀고, 입력한 이메일로 온 인증 코드를 입력하면 가입이 완료됩니다.
  2. Vercel 가입 (GitHub 연동)

    vercel.com에 접속해 우측 상단 "Sign Up"을 클릭합니다.

    1. 계정 유형에서 개인 실습용 "Hobby"를 선택하고 이름을 입력한 뒤 Continue합니다.
    2. "Continue with GitHub" 버튼을 클릭합니다.
    3. GitHub 로그인 화면이 뜨면 로그인하고, "Authorize Vercel"을 클릭해 연동을 허용합니다.
    4. Vercel 대시보드가 열리면 가입이 완료된 것입니다.
  3. OpenAI 결제 등록 및 API Key 발급

    platform.openai.com에 로그인(또는 가입)합니다. OpenAI API는 사용한 만큼 선불 크레딧에서 차감되는 방식이라,
    키를 발급하기 전에 먼저 결제 수단을 등록하고, 최소 금액을 충전해야 키가 정상 동작합니다.

    1. 좌측 하단(또는 우측 상단) 톱니바퀴 Settings → "Billing"으로 이동합니다.
    2. "Add payment details"(결제 수단 추가)를 클릭하고, 개인/법인 카드 정보를 입력해 등록합니다.
    3. 충전할 금액을 입력합니다 — 실습용으로는 최소 금액인 $5면 충분합니다.
    4. 자동 충전(정기 결제, "Enable auto recharge") 옵션은 반드시 체크 해제합니다.(잔액이 떨어질 때 카드에서 자동으로 다시 결제되는 것을 막기 위함입니다.)
    5. 결제를 완료하고 크레딧(잔액)이 충전되었는지 확인합니다.
    6. 좌측 메뉴에서 "API keys"로 이동해 "Create new secret key" 버튼을 클릭합니다.
    7. 키 이름을 입력하고 "Create secret key"를 클릭합니다.
    8. 표시된 sk-로 시작하는 키의 "Copy"를 눌러 복사한 뒤 안전한 곳에 보관하고 "Done"을 누릅니다.(이 화면을 벗어나면 키를 다시 볼 수 없습니다.)
API Key는 비밀번호와 같습니다. 코드·채팅·공개 저장소에 노출하지 마십시오. 발급 화면을 벗어나면 다시 볼 수 없으므로 반드시 복사해 두고, 유출이 의심되면 즉시 삭제(Revoke) 후 재발급하십시오. 또한 예기치 않은 과금을 막기 위해 자동 충전(auto recharge)은 꺼 두고, 충전한 $5 한도 안에서만 사용하십시오.
● 작업 도구 Toolchain AI 작업대(Claude Desktop)와 개발 도구(Node.js·Git)를 갖춥니다
STEP 2 설치 Claude 구독 + Claude Desktop 준비 (작업 폴더 생성)
이번 과정의 작업 도구인 Claude Desktop을 쓸 수 있게 만듭니다.
Claude를 Pro로 구독하고 앱을 설치하면, 작업 폴더(hello-page) 생성부터 파일 만들기·명령 실행까지 전부 채팅으로 할 수 있습니다. Claude Desktop은 내 컴퓨터의 파일과 명령을 이 채팅창에서 직접 다룰 수 있어, 따로 설치할 도구 없이 바로 시작합니다.(앞으로의 모든 실습은 이 Claude Desktop 채팅창에서 진행합니다.)
관련 이론
Claude·Claude Desktop의 이해
단계별 실행
  1. Claude 가입 및 유료 구독 (Pro 또는 Max)

    claude.ai에서 계정을 만든 뒤 아래 순서로 유료 플랜을 구독합니다.

    1. 로그인 후 화면 좌측 하단의 이름을 클릭하고 "Settings"를 선택합니다.
    2. "Billing" 탭으로 이동해 "Upgrade plan"을 클릭합니다.
    3. 실습 강도에 맞는 플랜을 선택하고 결제 주기를 고릅니다.(아래는 2026년 7월 기준 요금이며, 최신 금액은 결제 화면에서 확인하세요.)
      • Pro — 월 $20(연간 결제 시 약 $17/월). 개념 실습·PRD 작성 등 가벼운 작업에 적당합니다.
      • Max 5× — 월 $100. Pro의 약 5배 사용량. 본격 앱 제작 단계에 여유가 있습니다.
      • Max 20× — 월 $200. Pro의 약 20배 사용량. 장시간 집중 제작에 적합합니다.(Max는 월간 결제만 제공되며 연간 할인이 없습니다.)
    4. 카드 정보를 입력하고 "Subscribe"를 눌러 구독을 완료합니다.
    뒤 과정의 Claude Code는 파일을 읽고·쓰고·검증하는 작업을 반복해 토큰이 빠르게 소모됩니다. 고도화된 PRD 작성이나 앱 제작 같은 본격 단계에서는 Pro만으로는 사용량 한도에 금방 도달할 수 있습니다. 그래서 집중 제작 주간에는 Max를 권장하고, 이후 사용 빈도를 보며 Pro로 낮추는 방식을 추천합니다. 비용이 부담되면 Pro로 시작해도 되지만, 실습 중 한도에 걸리면 그날 진도가 끊길 수 있다는 점만 감안하세요.
  2. Claude Desktop 설치 및 로그인

    내 컴퓨터에 Claude 데스크톱 앱을 설치합니다.

    1. 브라우저에서 claude.ai/download 에 접속합니다.
    2. 운영체제에 맞는 설치 파일을 내려받습니다.
      • Windows: .exe 설치 파일
      • macOS: .dmg 파일 (Apple 실리콘·인텔 자동 감지)
    3. 내려받은 파일을 실행해 안내에 따라 설치를 완료합니다.
    4. Claude Desktop을 실행하고, 앞에서 유료 플랜을 구독한 계정으로 로그인합니다.
  3. 자연어로 작업 폴더(hello-page) 만들기

    폴더도 손으로 만들 필요 없이 Claude에게 시키면 됩니다. 새 대화창에 아래 프롬프트를 붙여넣으면, Claude Desktop이 실제로 폴더를 만들어 줍니다.

    프롬프트 
    바탕화면에 hello-page 라는 폴더를 직접 만들어줘.
폴더 이름은 영문·띄어쓰기 없이 정확히 hello-page 로 해줘.

    Claude가 명령을 실행해도 되는지 물으면 "허용(Allow)"을 누릅니다. 바탕화면에 hello-page 폴더가 생기면 성공입니다.(위치는 바탕화면이 가장 편하지만 어디든 상관없습니다. 다른 곳에 만들고 싶으면 "바탕화면에" 대신 원하는 경로를 알려주면 됩니다.)

    윈도우에서 바탕화면이 OneDrive로 백업되고 있어도 그대로 만들면 됩니다. 혹시 나중에 폴더를 못 찾거나 경로 관련 오류가 나면, 내 사용자 폴더(C:\Users\내이름)에 만들어 달라고 요청하면 가장 안전합니다.

Claude Desktop은 권한이 넓습니다. 파일뿐 아니라 명령까지 실행할 수 있어 컴퓨터 전반에 접근할 수 있으니, Claude가 파일 작업·명령을 실행하기 전에 뜨는 "허용" 팝업을 매번 확인하세요. 낯선 명령이 보이면 바로 허용하지 말고 무슨 명령인지 먼저 물어보는 것이 안전합니다.
앞으로의 모든 실습(STEP 4부터)은 이 Claude Desktop 채팅창에서 진행합니다. 대화창에 한국어로 지시하면, Claude Desktop이 hello-page 폴더에 파일을 직접 만들고 명령을 실행해 줍니다.
STEP 3 설치 Node.js·Git 설치를 터미널 없이 채팅으로
모든 실습의 토대인 Node.js와 Git을 준비합니다.
Claude Desktop이 명령을 직접 실행해 주므로, 이제 터미널을 전혀 열지 않고 Node.js 설치부터 Git 설치·버전 확인까지 전부 채팅으로 끝낼 수 있습니다.
관련 이론
Node.js·Git의 이해
단계별 실행
Node.js · Git 설치는 채팅으로 — 말로만, 터미널 없음
  1. Claude Desktop 채팅으로 설치 요청

    이제 대화창에 아래처럼 요청하면 Claude가 실제로 설치 명령을 대신 실행합니다.

    프롬프트 
    Node.js LTS 버전이랑 Git이 설치되어 있는지 확인해줘.
없으면 직접 설치해줘.
(Windows면 winget install OpenJS.NodeJS.LTS,
winget install Git.Git /
macOS면 brew install node git)

    Claude가 명령을 실행해도 되는지 물으면 "허용(Allow)"을 누릅니다.

  2. 설치 확인도 채팅으로

    이어서 아래처럼 요청하면 Claude가 직접 버전을 확인해 채팅창에 결과를 보여줍니다.

    프롬프트 
    node -v, npm -v, git --version을
각각 실행해서 결과를 알려줘.

    ▲ 예: v20.x.x, 10.x.x, git version 2.x.x

    "명령을 찾을 수 없다"는 결과가 나오면, Claude에게 "Claude Desktop을 재시작해야 하니 안내해줘"라고 요청하세요. 설치 직후에는 새로 여는 프로그램부터 인식되는 경우가 많습니다.

  3. Git 최초 설정도 채팅으로

    왜 하나요? — 코드를 저장(커밋)할 때마다 Git이 "누가 저장했는지"를 함께 기록합니다. 그래서 맨 처음 딱 한 번 내 이름·이메일을 Git에 알려 두면, 이후 저장할 때마다 자동으로 내 이름표가 붙습니다. (숙제에 미리 이름을 적어 두는 것과 같아요.) 이메일은 STEP 1에서 만든 GitHub 계정과 같게 맞추면, 나중에 GitHub에서도 내 기록으로 깔끔하게 연결됩니다.

    프롬프트 
    git config --global user.name을 "본인 이름"으로,
git config --global user.email을 "본인이메일@example.com"으로
직접 설정해줘.
앞으로도 계속 이런 식입니다. 파일을 만들고 고치는 일부터 프로그램 설치·실행까지, 내가 터미널에 직접 타이핑할 필요 없이 Claude에게 말로 부탁하면 대신 해 줍니다. STEP 5부터의 앱 만들기도 전부 이렇게 진행합니다.
Claude는 명령을 실행하기 전에 "이 명령을 실행할까요?" 하고 팝업으로 먼저 물어봅니다. 그때 어떤 명령인지 한 번 훑어보고 허용하세요. 무슨 명령인지 잘 모르겠거나 낯설면, 허용을 누르기 전에 Claude에게 "이 명령이 무슨 일을 하는지 알려줘"라고 물어보면 됩니다.

STEP 4 실습 헬로페이지 — 자연어로 만들고 고치기
코드를 한 줄도 쓰지 않고 한국어 지시(프롬프트)만으로 첫 웹페이지를 만듭니다.
STEP 2에서 만든 hello-page 폴더를 향해, Claude Desktop 대화창에 "이런 페이지를 만들어 줘"라고 말하는 것이 전부입니다.
관련 이론
바이브 코딩·프롬프트의 원리
단계별 실행
  1. Claude Desktop에서 새 대화 시작

    Claude Desktop을 실행하고 새 대화(New chat)를 엽니다. STEP 2에서 hello-page 폴더를 만들었으므로, Claude에게 그 폴더에서 작업해 달라고 하면 이 대화의 지시가 그 폴더에 반영됩니다.(폴더가 없으면 STEP 2의 "작업 폴더 만들기"를 먼저 확인하세요.)

  2. .env 파일 만들고 OpenAI API Key 넣기

    이후 실습에서 프로그램이 OpenAI를 호출할 때 쓸 API Key를 미리 준비합니다. 먼저 아래 프롬프트로 .env 파일을 만들어 달라고 요청합니다.

    프롬프트 ① 
    hello-page 폴더에 .env 파일을 직접 만들어줘.
그 안에 OPENAI_API_KEY 항목을 넣어줘.
그리고 .gitignore 파일에도 .env를 추가해서
API Key가 실수로 업로드되지 않게 해줘.

    Claude Desktop이 파일을 만들어도 되는지 물으면 "허용(Allow)"을 누릅니다. 이제 빈 .env 파일이 생겼으니, STEP 1에서 발급한 sk- 키를 이 파일에 넣을 차례입니다. 방법은 두 가지이며, 보안상 방법 A(내가 직접 넣기)를 권장합니다.

    ✅ 방법 A · 내가 직접 .env에 넣기 — 권장, 키가 채팅에 남지 않음
    1. hello-page 폴더에서 .env 파일을 찾아 마우스 오른쪽 → "연결 프로그램" → 메모장으로 엽니다.(파일 이름이 점(.)으로 시작해도 정상입니다. 더블클릭으로 안 열리면, 메모장을 먼저 켜고 파일 → 열기에서 오른쪽 아래를 "모든 파일"로 바꾼 뒤 .env를 선택하세요.)
    2. OPENAI_API_KEY= 뒤에 커서를 두고, 복사해 둔 sk- 키를 붙여넣기(Ctrl+V) 합니다. 최종적으로 OPENAI_API_KEY=sk-... 한 줄이 되어야 합니다.(등호(=) 앞뒤에 공백을 넣지 말고, 키를 따옴표(" ")로 감싸지도 마세요.)
    3. 저장(Ctrl+S) 한 뒤 메모장을 닫습니다. 키는 내 파일에만 저장되고 채팅에는 전혀 남지 않습니다.
    💬 방법 B · Claude에게 맡기기 — 간편하지만 키가 채팅 기록에 남음

관련 이론
배포(deploy)의 개념 Vercel의 이해

완성된 웹 페이지(index.html)를 인터넷에 배포합니다. 터미널 없이, GitHub와 Vercel 사이트에서 클릭만으로 진행합니다.

🌐 GitHub 업로드 후 Vercel 사이트에서 배포 — 이후 자동 재배포
  1. GitHub에 새 저장소(repository) 만들기

    STEP 1에서 만든 GitHub 계정으로 로그인한 뒤, 코드를 올려 둘 빈 저장소를 하나 만듭니다.

    1. github.com 우측 상단의 "+" 아이콘을 클릭하고 "New repository"를 선택합니다.
    2. Repository name에 hello-page를 입력합니다.
    3. 공개 범위는 "Public"을 선택합니다.(README 등 다른 항목은 체크하지 않고 비워 둡니다.)
    4. "Create repository"를 클릭합니다. 다음 화면에 나오는 저장소 주소(https://github.com/사용자명/hello-page.git)를 복사해 둡니다.
  2. 결과물을 GitHub에 업로드하기

    터미널 없이 GitHub 사이트에서 파일을 바로 올릴 수 있습니다. 방금 만든 빈 저장소 화면에서 진행합니다.

    1. 저장소 화면의 "Add file" → "Upload files"를 클릭합니다.
    2. hello-page 폴더에서 index.html·재정경제부.svg를 끌어다 놓거나 "choose your files"로 선택합니다.(자기소개·시계만 있는 정적 페이지라 이 두 파일이면 됩니다.)
    3. 아래 "Commit changes" 버튼을 눌러 업로드를 확정합니다.
    4. 저장소에 파일들이 올라온 것을 확인합니다.
  3. Vercel 사이트에서 저장소 연결·배포

    vercel.com 대시보드에서 GitHub 저장소를 그대로 불러와 배포합니다.

    1. vercel.com에 로그인하고 "Add New…" → "Project"를 클릭합니다.
    2. GitHub 저장소 목록에서 hello-page를 찾아 "Import"를 클릭합니다.(목록에 없으면 "Adjust GitHub App Permissions"로 저장소 접근을 허용합니다.)
    3. 설정은 기본값 그대로 두고 "Deploy"를 클릭합니다.
    4. 잠시 기다리면 배포가 완료되고 축하 화면과 함께 공개 주소가 나타납니다.
  4. 공개 주소 접속

    Vercel이 만들어 준 https://hello-page-xxxx.vercel.app 주소로 접속해, 내가 만든 페이지가 인터넷에 공개된 것을 확인합니다. 이 주소는 다른 사람에게 공유해도 그대로 접속됩니다.

    이 방식은 GitHub 저장소와 연결되어 있어, 앞으로 코드를 수정해 GitHub에 다시 올리기만 하면 Vercel이 자동으로 다시 배포합니다.
Vercel은 GitHub에 올린 파일을 인터넷상의 자기 저장 공간에 올리고, 누구나 접속할 수 있는 고유 주소를 자동으로 만들어 줍니다. 한 번 연결해 두면, 앞으로 코드를 고쳐 GitHub에 다시 올리기만 하면 Vercel이 자동으로 다시 배포합니다.
정리: "자연어로 만들고 → 고치고 → 세상에 공개(배포)"까지 한 바퀴를 돌았습니다. 다음 STEP 7부터는 이 흐름 위에 AI 챗봇(RAG·멀티세션)을 얹어, 문서를 기억하고 대화를 저장하는 앱으로 키웁니다.
● 멀티세션 RAG Multi-Session 문서를 근거로 답하고, 대화를 저장·불러오는 RAG 챗봇 (Supabase)
STEP 7 실습 📚 멀티세션 RAG 챗봇 — 문서를 기억하고 대화를 저장하기
STEP 6까지 만든 페이지는 대화를 나눌 수 없습니다. 이번에는 본격적인 AI 챗봇을 만드는데, 그냥 챗봇이 아니라 두 가지를 갖춥니다. ① 내 PDF 문서를 올려 그 내용을 근거로 답하는 RAG, ② 대화를 클라우드 DB에 저장해 껐다 켜도 이어서 볼 수 있는 멀티세션. 파이썬·Streamlit 없이, 지금까지처럼 정적 index.html로 화면을 만들고, 키를 안전하게 다루는 작은 api/*.js 연결 파일과 저장용 Supabase(무료 클라우드 DB)를 더합니다. STEP 4에서 만든 헬로페이지 폴더(hello-page)에서 이어서 진행합니다 — 새 폴더를 따로 만들지 않고, 지금까지의 index.html(자기소개·시계) 아래에 챗봇 섹션을 이어 붙입니다.
Q. RAG가 뭐죠? 왜 DB가 필요하죠?
A. RAG는 내 문서를 잘게 쪼개 '임베딩(벡터)'이라는 숫자 묶음으로 바꿔 DB에 저장해 두고, 질문이 오면 질문과 가장 비슷한 조각들을 찾아 그 내용을 근거로 답하게 하는 방식입니다. 세션은 대화 한 묶음(제목 + 주고받은 메시지들)을 말하며, 이걸 DB에 저장해 두면 나중에 목록에서 골라 다시 불러올 수 있습니다. 대화와 문서 벡터를 모두 담아 둘 곳이 필요해서 index.html 뒤에 Supabase를 두는 것입니다. (브라우저는 DB 키를 읽지 못하므로, 화면 뒤에서 도는 api/*.js가 대신 접근합니다.)

● 멀티유저 Multi-User 로그인해서 사용자별로 세션·문서를 분리합니다
STEP 8 실습 👥 멀티유저 — 로그인해서 내 세션·문서만 보기
STEP 7의 멀티세션 앱은 모두가 같은 대화·문서를 공유합니다. 이번에는 로그인을 더해, 사용자마다 자기 세션·문서만 보이도록 나눕니다. Supabase의 로그인 기능(Auth)은 쓰지 않고, 우리가 직접 만든 user 테이블로 회원가입/로그인을 처리합니다. 비밀번호는 반드시 해시(암호화)로 저장하고, 모든 조회·저장·삭제는 로그인한 사용자(user_id) 기준으로만 이뤄지게 합니다. STEP 7에서 만든 hello-page의 챗봇을 그대로 확장하며, 챗봇 이름은 "ENA RAG 챗봇"으로 바꿉니다.
Q. 비밀번호를 그냥 저장하면 안 되나요?
A. 절대 안 됩니다. 비밀번호를 평문(그대로)으로 저장하면, DB가 유출될 때 모든 사용자의 비밀번호가 그대로 새어 나갑니다. 그래서 되돌릴 수 없는 해시로 바꿔 저장하고, 로그인할 때는 입력값을 같은 방식으로 해시해서 비교만 합니다. 이 실습에서도 해시 저장을 필수로 요청합니다.

① 비밀번호는 반드시 해시로 저장하고 평문으로 두지 마세요. ② 모든 데이터 접근은 user_id 조건을 빠뜨리지 마세요(빠지면 남의 데이터가 보입니다). ③ 공개 배포 시 키는 .env가 아니라 Vercel 프로젝트 환경 변수(OPENAI_API_KEY·SUPABASE_URL·SUPABASE_ANON_KEY)에 등록하세요. ④ 로컬은 .env, 배포는 Vercel 환경 변수 — 두 곳을 같은 값으로 맞추면 됩니다.
오늘의 확장: "정적 페이지 → 챗봇 → RAG·멀티세션 → 멀티유저"까지, 코드를 직접 쓰지 않고 자연어 프롬프트만으로 실제 서비스 형태의 앱을 완성했습니다.
