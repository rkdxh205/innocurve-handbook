● 초급 Getting Started 텔레그램 앱에서 직접 BotFather와 대화하며 나만의 봇을 처음 만들어 봅니다
G-0 초급 Telegram Desktop 설치하고 GitHub 계정까지 미리 만들어두기
아직 텔레그램 앱조차 설치하지 않았습니다. 오늘 실습은 노트북(데스크톱)만으로 진행하므로 스마트폰 앱 설치는 건너뛰고 Telegram Desktop으로 바로 가입합니다. 이후 실습에서 코드를 저장하려면 GitHub 계정도 필요하므로, 인증에 시간이 걸릴 수 있는 두 가지를 본격적인 봇 만들기에 앞서 먼저 끝내둡니다.
사용 요소
Telegram Desktop 설치 GitHub 계정 만들기
단계별 실행
  1. 1. Telegram Desktop 설치

    telegram.org/apps(또는 Microsoft Store)에서 "Telegram Desktop"을 내려받아 설치합니다. 스마트폰 앱은 설치하지 않습니다.

  2. 2. 전화번호로 가입

    데스크톱 앱을 실행해 휴대폰 번호를 입력하면 문자(SMS)로 인증 코드가 도착합니다. 이 코드를 앱 화면에 입력하면 가입이 끝납니다 — 스마트폰에 텔레그램 앱이 없어도 문자만 받으면 됩니다.

  3. 3. GitHub 계정 가입

    github.com에서 Sign up을 눌러 이메일·비밀번호·username(영문 권장)을 입력하고 인증을 마쳐 계정을 만듭니다.

GitHub 계정으로 실제로 무엇을 하게 될지는 이후 실습에서 안내합니다. 오늘은 로그인 가능한 계정을 만들어 두는 것까지만 완료하면 충분합니다. 그리고 봇이 완성된 뒤 스마트폰에도 같은 계정으로 텔레그램 앱을 설치해두면, 외출 중에도 편하게 뉴스 알림을 받아볼 수 있습니다.
G-1 초급 내 이름의 텔레그램 봇을 직접 만들고 토큰 발급받기
아직 텔레그램 봇을 한 번도 만들어 본 적이 없습니다. 코드를 작성하기 전에, 먼저 텔레그램 앱 안에서 손으로 직접 대화하며 나만의 봇 계정 하나를 만들고, 앞으로 실습에서 계속 쓸 봇 토큰을 발급받아 안전하게 메모해 두려고 합니다.
사용 요소
BotFather 봇 토큰
단계별 실행
  1. @BotFather와 대화 시작하기

    텔레그램 앱 상단 검색창에 @BotFather를 입력해 공식 계정을 찾고, 대화방에 들어가 시작 버튼을 누릅니다.

  2. /newbot 전송하고 이름 정하기

    대화창에 /newbot을 입력해 전송합니다. BotFather가 먼저 표시 이름을 물으면 원하는 이름을 입력합니다.

    /newbot
  3. username 정하기

    이어서 username을 물으면, 5~32자·영문/숫자/밑줄만 사용하고 반드시 "bot"으로 끝나는 이름을 입력합니다. 한 번 정하면 바꿀 수 없으니 신중하게 고릅니다.

    예시 username: gildong_news_alert_bot
  4. 발급된 토큰을 안전하게 메모하기

    등록이 완료되면 BotFather가 곧바로 봇 토큰을 대화창에 보여줍니다. 이 값을 그대로 복사해 본인만 아는 메모장(또는 비밀번호 관리 앱)에 저장합니다. 절대 공개된 곳에 붙여넣지 않습니다.

    <여기에_발급받은_BOT_TOKEN>
BotFather에게 /newbot을 보내면, 텔레그램 서버 내부에 새 봇 계정이 등록되고 그 봇만의 고유 ID와 비밀 토큰이 함께 생성됩니다. 이 토큰이 있어야만 프로그램이 그 봇의 이름으로 메시지를 보내거나 받을 수 있습니다.
표시 이름은 나중에 BotFather의 /setname 명령으로 바꿀 수 있지만, username은 절대 바꿀 수 없습니다. 처음부터 알아보기 쉬운 이름(예: 본인 성함 이니셜 + news + bot)으로 정해 두면 나중에 헷갈리지 않습니다.
● 중급 Daily Workflow Claude Code에게 요청해 파이썬 코드로 내 봇이 실제 메시지를 보내게 만들어 봅니다
D-1 중급 Claude Code와 함께 파이썬으로 "안녕하세요" 메시지 보내기
G-1에서 봇과 토큰을 이미 발급받았습니다. 이번에는 텔레그램 앱을 직접 조작하지 않고, Claude Code에게 요청해서 파이썬 코드가 대신 내 봇에게 "안녕하세요" 메시지를 보내도록 만들려고 합니다. 그러려면 먼저 내 대화방 번호(chat_id)를 알아내야 합니다.
사용 요소
sendMessage getUpdates
단계별 실행
  1. 내 봇에게 먼저 말 걸기

    텔레그램에서 G-1에서 만든 내 봇을 검색해 대화방을 열고, 아무 텍스트나 한 번 보냅니다. (예: "안녕") 봇은 아직 응답하지 않지만, 이 메시지가 다음 단계에서 chat_id를 확인하는 데 쓰입니다.

  2. Claude Code에게 chat_id 확인 스크립트 요청하기

    Claude Code에게 아래 프롬프트를 그대로 복사해 붙여넣습니다.

    Claude Code 프롬프트
    requests 라이브러리로 getUpdates를 호출해서 chat_id를 출력하는 파이썬 스크립트를 짜줘

    아래와 같은 코드가 만들어지고 나면 직접 실행해 결과에 표시되는 chat.id 값을 메모합니다.

    import requests

TOKEN = "<여기에_발급받은_BOT_TOKEN>"
url = f"https://api.telegram.org/bot{TOKEN}/getUpdates"

response = requests.get(url, timeout=15)
data = response.json()

for update in data.get("result", []):
    message = update.get("message", {})
    print("chat_id:", message.get("chat", {}).get("id"))
    print("text:", message.get("text"))
  3. Claude Code에게 sendMessage 스크립트 요청하기

    이어서 아래 프롬프트를 그대로 복사해 요청합니다.

    Claude Code 프롬프트
    이 chat_id로 '안녕하세요' 메시지를 보내는 sendMessage 스크립트도 짜줘

    실행하면 내 텔레그램 대화방에 실제로 메시지가 도착합니다.

    import requests

TOKEN = "<여기에_발급받은_BOT_TOKEN>"
CHAT_ID = "<1단계에서_확인한_chat_id>"
url = f"https://api.telegram.org/bot{TOKEN}/sendMessage"

requests.post(url, json={
    "chat_id": CHAT_ID,
    "text": "안녕하세요"
}, timeout=15)
requests.get과 requests.post는 각각 https://api.telegram.org/bot<토큰>/METHOD_NAME 주소로 인터넷 요청을 보냅니다. 텔레그램 서버는 토큰으로 어느 봇의 요청인지 확인한 뒤, getUpdates는 쌓인 메시지 목록을, sendMessage는 발송 성공 여부를 응답으로 돌려줍니다.
토큰을 코드 안에 그대로 적어 GitHub 같은 공개 저장소에 올리지 않도록 주의합니다. 이후 자동화 실습에서는 토큰을 코드가 아닌 별도의 비밀 값(secret) 저장소에 넣는 방법을 다룹니다.
D-2 중급 네이버 오픈 API 키 발급받고 테스트 호출하기
텔레그램 봇은 준비됐지만, 뉴스를 검색해올 열쇠가 아직 없습니다. 네이버 개발자센터에서 뉴스검색 API용 Client ID·Client Secret을 발급받고, Claude Code와 함께 정말로 작동하는지 간단히 테스트 호출까지 해봅니다.
사용 요소
네이버 오픈 API 등록
단계별 실행
  1. 1. 네이버 개발자센터에서 애플리케이션 등록

    developers.naver.com에 로그인해 Application → 애플리케이션 등록으로 들어가, 이름을 입력하고 이용 API에서 "검색(Search)"을 체크합니다. 서비스 환경은 WEB, 서비스 URL은 http://localhost로 입력해도 됩니다.

  2. 2. Client ID·Client Secret 확인 및 메모

    등록이 끝나면 애플리케이션 상세 화면에 표시되는 Client ID와 Client Secret을 봇 토큰과 마찬가지로 안전한 곳에 메모해둡니다.

  3. 3. Claude Code에게 테스트 호출 스크립트 요청하기
    Claude Code 프롬프트
    발급받은 네이버 Client ID·Secret으로 뉴스검색 API를 호출해서 검색 결과 개수를 출력하는 파이썬 스크립트를 짜줘
    import requests

CLIENT_ID = "<여기에_발급받은_CLIENT_ID>"
CLIENT_SECRET = "<여기에_발급받은_CLIENT_SECRET>"

headers = {
    "X-Naver-Client-Id": CLIENT_ID,
    "X-Naver-Client-Secret": CLIENT_SECRET,
}
url = "https://openapi.naver.com/v1/search/news.json?query=테스트"

response = requests.get(url, headers=headers, timeout=15)
data = response.json()
print("검색 결과 개수:", len(data.get("items", [])))
  4. 4. 정상 동작 확인

    실행 결과로 오류 없이 숫자가 출력되면 Client ID·Secret이 정상적으로 발급·연결된 것입니다. 실제로 이 키로 뉴스를 검색하고 필터링·전송까지 하는 완성된 로직은 바로 다음 P-2에서 이어집니다.

네이버 뉴스검색 API도 텔레그램 Bot API와 마찬가지로, 발급받은 인증 값을 요청에 실어 보내면 서버가 그 값을 확인한 뒤 결과를 돌려주는 구조입니다. 다만 텔레그램은 URL 경로에 토큰을 넣고, 네이버는 HTTP 헤더에 Client ID·Secret을 넣는다는 점이 다릅니다.
Client Secret 역시 봇 토큰과 동일하게 취급해야 하는 비밀 값입니다. 코드에 직접 적지 말고 별도의 비밀 값 저장 방식을 쓰는 습관을 지금부터 들이는 것이 좋습니다.
● 고급 Power User 명령어 응답 로직 설계 → 뉴스 검색·필터링·중복 방지로 봇 완성 → GitHub Actions로 완전 자동화까지 진행합니다
P-1 고급 Claude Code와 함께 /start, /status 명령어에 응답하는 로직 설계하기
지금까지는 봇이 항상 같은 메시지만 보냈습니다. 이번에는 사용자가 /start나 /status 같은 명령어를 보냈을 때, 봇이 그 내용을 구분해서 서로 다르게 응답하도록 Claude Code와 함께 로직을 설계하려고 합니다. 이런 명령어 분기 방식은 이후 만들 다른 에이전트에도 그대로 응용됩니다.
사용 요소
getUpdates sendMessage offset
단계별 실행
  1. Claude Code에게 요청 내용 설명하기
    Claude Code 프롬프트
    getUpdates로 받은 메시지의 text가 /start인지 /status인지 확인해서, 각각 다른 안내 문구를 보내는 파이썬 함수를 짜줘. 그리고 한 번 처리한 메시지를 중복해서 또 처리하지 않도록 offset도 관리해줘
  2. 명령어 분기 로직 확인하기

    Claude Code가 아래와 비슷한 형태의 코드를 만들어 줍니다. tg_get은 Bot API의 GET 메서드를, send_message는 sendMessage를 감싼 보조 함수이고, offset은 이미 처리한 메시지의 update_id 바로 다음 값으로 갱신해 같은 메시지를 두 번 처리하지 않게 합니다.

    import requests

TOKEN = "<여기에_발급받은_BOT_TOKEN>"
TGAPI = f"https://api.telegram.org/bot{TOKEN}"

def tg_get(method, **params):
    r = requests.get(f"{TGAPI}/{method}", params=params, timeout=15)
    r.raise_for_status()
    return r.json()

def send_message(chat_id, text):
    requests.post(f"{TGAPI}/sendMessage", json={"chat_id": chat_id, "text": text}, timeout=15)

offset = None

def handle_updates():
    global offset
    resp = tg_get("getUpdates", offset=offset, timeout=0)
    for upd in resp.get("result", []):
        offset = upd["update_id"] + 1
        msg = upd.get("message", {})
        text = msg.get("text", "").strip()
        chat_id = msg.get("chat", {}).get("id")

        if text == "/start":
            send_message(chat_id, "안녕하세요! 뉴스 알림 봇입니다.")
        elif text == "/status":
            send_message(chat_id, "정상 작동 중입니다. 뉴스가 검색되면 알려드립니다.")
  3. 명령어 추가로 확장해 보기

    같은 방식으로 elif 구문을 추가하면 /help 같은 명령어도 얼마든지 늘릴 수 있습니다.

    Claude Code 프롬프트
    명령어를 하나 더 추가해줘
getUpdates가 반환하는 각 항목에는 고유한 update_id가 붙어 있습니다. 처리할 때마다 offset을 그 값+1로 올려두면, 다음 호출에서는 이미 처리한 메시지를 다시 받지 않습니다. 이 반복 호출 구조가 이후 GitHub Actions의 정기 실행(cron)과 결합되면, 사람이 지켜보지 않아도 새 메시지가 올 때마다 자동으로 응답하는 봇이 됩니다.
지금은 코드를 직접 실행해야만 봇이 응답하지만, 이후 자동화 실습에서 이 로직을 주기적으로 자동 실행하도록 설정하면 사람이 켜 두지 않아도 항상 대기 중인 봇이 됩니다.
함께 보기: 이어서 P-2에서 실제 뉴스 검색·필터링·중복 방지 로직까지 합쳐 오늘 안에 봇을 완성합니다. 이 봇을 사람이 켜두지 않아도 저절로 실행되게 만드는 자동화는 다음 Day에서 새로운 주제로 이어집니다.
P-2 고급 이름·관심 키워드 검색, 시간범위, 동명이인 필터링, 중복 방지를 합쳐 진짜 동작하는 봇 완성하기
지금까지 봇 토큰과 네이버 API 키를 발급받고, 메시지를 보내고 받는 법도 익혔습니다. 이제 D-2에서 테스트했던 네이버 뉴스검색 API 호출에 "내 이름이 맞는지 걸러내기", "관심 키워드 기사도 함께 받기", "몇 시간 이내 기사까지 받을지 정하기", "이미 보낸 기사 제외하기"를 모두 더해, 실제로 매번 실행할 때마다 원하는 뉴스만 골라 텔레그램으로 보내주는 완성된 봇을 오늘 안에 만들어봅니다.
사용 요소
뉴스검색 API RECENT_HOURS 동명이인 필터 관심 키워드 필터 중복 방지 sendMessage
단계별 실행
  1. 1. Claude Code에게 전체 로직 요청하기
    Claude Code 프롬프트
    내 이름 조합 검색어와 관심 키워드 검색어로 네이버 뉴스 API를 각각 검색하고, 최근 N시간 이내 기사만 남긴 뒤, 이름 기사는 동명이인 필터로·관심 키워드 기사는 포함 여부로 걸러내고, seen_urls.txt에 없는 새 기사만 텔레그램으로 보낸 뒤 seen_urls.txt를 갱신하는 파이썬 스크립트를 짜줘
  2. 2. 완성된 스크립트 확인하기

    Claude Code가 검색 → 시간 필터 → 이름/키워드 필터 → 중복 제거 → 전송을 하나로 묶어 아래와 비슷한 형태의 스크립트를 만들어 줍니다.

    import re, html, os, requests
from datetime import datetime, timezone, timedelta
from email.utils import parsedate_to_datetime

BOT_TOKEN = "<여기에_발급받은_BOT_TOKEN>"
NAVER_ID = "<여기에_발급받은_CLIENT_ID>"
NAVER_SECRET = "<여기에_발급받은_CLIENT_SECRET>"
CHAT_ID = "<1단계에서_확인한_chat_id>"

TARGET_NAME = "홍길동"
NAME_QUERIES = ["홍길동 재정경제부", "홍길동 세종"]        # 나에 대한 뉴스
INTEREST_KEYWORDS = ["경제", "주식"]                       # 내가 관심있는 주제 뉴스
REQUIRED_KEYWORDS = ["세종", "세종시", "재정경제부"]
EXCLUDE_CONTEXTS = ["배우", "가수", "교수"]
RECENT_HOURS = 3          # 몇 시간 이내 기사까지 받을지 (작을수록 실시간에 가까움)
SEEN_FILE = "seen_urls.txt"

def is_target_person(text):
    if TARGET_NAME not in text:
        return False
    if not any(k in text for k in REQUIRED_KEYWORDS):
        return False
    if any(k in text for k in EXCLUDE_CONTEXTS):
        return False
    return True

def is_interest_match(text, keyword):
    return keyword in text

def search_naver(query):
    headers = {"X-Naver-Client-Id": NAVER_ID, "X-Naver-Client-Secret": NAVER_SECRET}
    url = f"https://openapi.naver.com/v1/search/news.json?query={requests.utils.quote(query)}&display=20&sort=date"
    return requests.get(url, headers=headers, timeout=15).json().get("items", [])

def fetch_articles():
    cutoff = datetime.now(timezone.utc) - timedelta(hours=RECENT_HOURS)
    articles = []

    for query in NAME_QUERIES:
        for item in search_naver(query):
            title = html.unescape(re.sub(r"<[^>]+>", "", item["title"]))
            desc = html.unescape(re.sub(r"<[^>]+>", "", item["description"]))
            pub_dt = parsedate_to_datetime(item["pubDate"])
            if pub_dt < cutoff or not is_target_person(title + " " + desc):
                continue
            articles.append({"title": title, "url": item["originallink"] or item["link"]})

    for keyword in INTEREST_KEYWORDS:
        for item in search_naver(keyword):
            title = html.unescape(re.sub(r"<[^>]+>", "", item["title"]))
            desc = html.unescape(re.sub(r"<[^>]+>", "", item["description"]))
            pub_dt = parsedate_to_datetime(item["pubDate"])
            if pub_dt < cutoff or not is_interest_match(title + " " + desc, keyword):
                continue
            articles.append({"title": title, "url": item["originallink"] or item["link"]})

    return articles

def load_seen():
    if not os.path.exists(SEEN_FILE):
        return set()
    with open(SEEN_FILE, encoding="utf-8") as f:
        return {line.strip() for line in f if line.strip()}

def save_seen(seen):
    with open(SEEN_FILE, "w", encoding="utf-8") as f:
        f.write("\n".join(sorted(seen)))

def send_message(text):
    requests.post(f"https://api.telegram.org/bot{BOT_TOKEN}/sendMessage",
                  json={"chat_id": CHAT_ID, "text": text}, timeout=15)

seen = load_seen()
new_count = 0
for article in fetch_articles():
    if article["url"] in seen:
        continue
    send_message(f"🔔 새 뉴스 알림\n\n📌 {article['title']}\n\n🔗 {article['url']}")
    seen.add(article["url"])
    new_count += 1

save_seen(seen)
print(f"신규 {new_count}건 전송 완료")
  3. 3. 직접 실행해서 동작 확인하기

    이 스크립트를 로컬에서 실행하면 내 이름이 들어간 최신 기사와 관심 키워드 기사가 함께 텔레그램으로 도착합니다. RECENT_HOURS를 줄이거나 늘려 실행해보면 더 최근 기사만/더 넓은 범위의 기사까지 받아지는 차이도 확인할 수 있습니다. 다시 한 번 실행하면 방금 받은 기사는 seen_urls.txt 덕분에 다시 오지 않습니다.

검색(이름 쿼리 + 관심 키워드 쿼리) → 시간 필터(RECENT_HOURS) → 내용 필터(is_target_person / is_interest_match) → 중복 제거(seen_urls) → 전송(sendMessage)이 한 스크립트 안에서 순서대로 실행됩니다. 이 다섯 단계가 바로 오늘 처음부터 끝까지 만든 "내 이름 + 관심 키워드 뉴스 알림 봇"의 전체 동작 원리입니다.
검색어를 여러 개(예: 이름+소속, 이름+지역, 관심 주제 여러 개) 조합해 NAME_QUERIES·INTEREST_KEYWORDS 리스트에 추가하도록 Claude Code에게 요청하면, 더 폭넓게 기사를 수집하면서도 필터 로직은 그대로 재사용할 수 있습니다.
함께 보기: 이어서 P-3에서 이 스크립트를 GitHub에 올려, 사람이 실행하지 않아도 저절로 하루 2번 자동 실행되게 만듭니다.
P-3 고급 GitHub Actions로 하루 2번 자동 실행되게 만들기
P-2에서 완성한 스크립트는 아직 내가 직접 실행해야만 동작합니다. 이번에는 이 코드를 GitHub 저장소에 올리고, GitHub Actions 워크플로를 설정해서 내 컴퓨터를 꺼두어도 매일 아침·저녁 두 번씩 서버가 대신 실행하고, 실행 결과(seen_urls.txt)까지 저장소에 자동으로 반영되도록 만듭니다.
사용 요소
GitHub Secrets GitHub Actions cron workflow_dispatch
단계별 실행
  1. 1. 하드코딩된 값을 환경변수로 바꾸기

    코드 안에 비밀 값이 그대로 남지 않도록, 아래 프롬프트로 요청해 바꿉니다.

    Claude Code 프롬프트
    이 스크립트의 BOT_TOKEN, NAVER_ID, NAVER_SECRET, CHAT_ID를 os.environ으로 읽도록 바꿔줘
  2. 2. GitHub 저장소를 만들고 코드 올리기

    github.com에서 Private 저장소를 하나 만들고, Claude Code에게 아래 프롬프트를 요청합니다.

    Claude Code 프롬프트
    이 폴더를 방금 만든 저장소에 올려줘

    .env 파일은 .gitignore에 추가해 절대 함께 올라가지 않도록 합니다.

  3. 3. Secrets 등록하기

    저장소의 Settings → Secrets and variables → Actions에서 BOT_TOKEN, NAVER_CLIENT_ID, NAVER_CLIENT_SECRET, CHAT_ID 네 가지를 각각 등록합니다.

  4. 4. 워크플로 파일 요청하기
    Claude Code 프롬프트
    check_and_send.py를 매일 KST 오전 8시·오후 6시에 실행하고, 실행 후 seen_urls.txt 변경사항을 자동으로 커밋·푸시하는 GitHub Actions 워크플로 yml을 만들어줘
    name: 내 이름 뉴스 알림 봇

on:
  schedule:
    - cron: '0 23 * * *'   # 매일 오전 8시 KST (UTC 23:00)
    - cron: '0 9 * * *'    # 매일 오후 6시 KST (UTC 09:00)
  workflow_dispatch:

concurrency:
  group: news-check
  cancel-in-progress: false

jobs:
  check:
    runs-on: ubuntu-latest
    permissions:
      contents: write
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.12'
      - run: pip install requests
      - name: 뉴스 체크 및 전송
        env:
          BOT_TOKEN: ${{ secrets.BOT_TOKEN }}
          NAVER_CLIENT_ID: ${{ secrets.NAVER_CLIENT_ID }}
          NAVER_CLIENT_SECRET: ${{ secrets.NAVER_CLIENT_SECRET }}
          CHAT_ID: ${{ secrets.CHAT_ID }}
        run: python check_and_send.py
      - name: seen_urls.txt 커밋
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add seen_urls.txt
          if git diff --cached --quiet; then
            echo "변경 없음"
          else
            git commit -m "Update seen_urls"
            git pull --rebase -X theirs origin main
            git push origin main
          fi
  5. 5. 수동 실행으로 바로 테스트하기

    cron 시각까지 기다릴 필요 없이, 저장소의 Actions 탭 → 워크플로 선택 → Run workflow 버튼으로 즉시 실행해볼 수 있습니다. 실행 로그와 실제 텔레그램 메시지 도착 여부를 함께 확인합니다.

GitHub Actions는 정해진 cron 시각마다(또는 수동 클릭 시) 가상 서버 하나를 새로 띄워 저장소 코드를 내려받고, Secrets 값을 환경변수로 주입해 스크립트를 실행합니다. 실행이 끝나면 마지막 스텝이 변경된 seen_urls.txt를 저장소에 다시 커밋해, 다음 실행에서도 중복 방지가 그대로 이어지게 만듭니다.
concurrency 설정이 없으면 실행 시간이 밀릴 때 이전 실행과 다음 실행이 겹쳐 같은 기사를 두 번 보낼 수 있습니다. 반드시 함께 설정합니다.
여기까지 오면: 내 컴퓨터를 켜두지 않아도 하루 2번씩 자동으로 내 이름·관심 키워드 뉴스를 찾아 텔레그램으로 보내주는 봇이 완성됩니다 — Day 2의 최종 결과물입니다.
