텔레그램 봇 만들기 실습 예시

● 초급 Getting Started 텔레그램 앱에서 직접 BotFather와 대화하며 나만의 봇을 처음 만들어 봅니다
G-1 초급 내 이름의 텔레그램 봇을 직접 만들고 토큰 발급받기
아직 텔레그램 봇을 한 번도 만들어 본 적이 없습니다. 코드를 작성하기 전에, 먼저 텔레그램 앱 안에서 손으로 직접 대화하며 나만의 봇 계정 하나를 만들고, 앞으로 실습에서 계속 쓸 봇 토큰을 발급받아 안전하게 메모해 두려고 합니다.
사용 요소
BotFather 봇 토큰
단계별 실행
  1. @BotFather와 대화 시작하기

    텔레그램 앱 상단 검색창에 @BotFather를 입력해 공식 계정을 찾고, 대화방에 들어가 시작 버튼을 누릅니다.

  2. /newbot 전송하고 이름 정하기

    대화창에 /newbot을 입력해 전송합니다. BotFather가 먼저 표시 이름을 물으면 원하는 이름을 입력합니다.

    /newbot
  3. username 정하기

    이어서 username을 물으면, 5~32자·영문/숫자/밑줄만 사용하고 반드시 "bot"으로 끝나는 이름을 입력합니다. 한 번 정하면 바꿀 수 없으니 신중하게 고릅니다.

    예시 username: gildong_news_alert_bot
  4. 발급된 토큰을 안전하게 메모하기

    등록이 완료되면 BotFather가 곧바로 봇 토큰을 대화창에 보여줍니다. 이 값을 그대로 복사해 본인만 아는 메모장(또는 비밀번호 관리 앱)에 저장합니다. 절대 공개된 곳에 붙여넣지 않습니다.

    <여기에_발급받은_BOT_TOKEN>
BotFather에게 /newbot을 보내면, 텔레그램 서버 내부에 새 봇 계정이 등록되고 그 봇만의 고유 ID와 비밀 토큰이 함께 생성됩니다. 이 토큰이 있어야만 프로그램이 그 봇의 이름으로 메시지를 보내거나 받을 수 있습니다.
표시 이름은 나중에 BotFather의 /setname 명령으로 바꿀 수 있지만, username은 절대 바꿀 수 없습니다. 처음부터 알아보기 쉬운 이름(예: 본인 성함 이니셜 + news + bot)으로 정해 두면 나중에 헷갈리지 않습니다.
● 중급 Daily Workflow Claude Code에게 요청해 파이썬 코드로 내 봇이 실제 메시지를 보내게 만들어 봅니다
D-1 중급 Claude Code와 함께 파이썬으로 "안녕하세요" 메시지 보내기
G-1에서 봇과 토큰을 이미 발급받았습니다. 이번에는 텔레그램 앱을 직접 조작하지 않고, Claude Code에게 요청해서 파이썬 코드가 대신 내 봇에게 "안녕하세요" 메시지를 보내도록 만들려고 합니다. 그러려면 먼저 내 대화방 번호(chat_id)를 알아내야 합니다.
사용 요소
sendMessage getUpdates
단계별 실행
  1. 내 봇에게 먼저 말 걸기

    텔레그램에서 G-1에서 만든 내 봇을 검색해 대화방을 열고, 아무 텍스트나 한 번 보냅니다. (예: "안녕") 봇은 아직 응답하지 않지만, 이 메시지가 다음 단계에서 chat_id를 확인하는 데 쓰입니다.

  2. Claude Code에게 chat_id 확인 스크립트 요청하기

    Claude Code에게 "requests 라이브러리로 getUpdates를 호출해서 chat_id를 출력하는 파이썬 스크립트를 짜줘"라고 요청합니다. 아래와 같은 코드가 만들어지고 나면 직접 실행해 결과에 표시되는 chat.id 값을 메모합니다.

    import requests

TOKEN = "<여기에_발급받은_BOT_TOKEN>"
url = f"https://api.telegram.org/bot{TOKEN}/getUpdates"

response = requests.get(url, timeout=15)
data = response.json()

for update in data.get("result", []):
    message = update.get("message", {})
    print("chat_id:", message.get("chat", {}).get("id"))
    print("text:", message.get("text"))
  3. Claude Code에게 sendMessage 스크립트 요청하기

    이어서 "이 chat_id로 '안녕하세요' 메시지를 보내는 sendMessage 스크립트도 짜줘"라고 요청합니다. 실행하면 내 텔레그램 대화방에 실제로 메시지가 도착합니다.

    import requests

TOKEN = "<여기에_발급받은_BOT_TOKEN>"
CHAT_ID = "<1단계에서_확인한_chat_id>"
url = f"https://api.telegram.org/bot{TOKEN}/sendMessage"

requests.post(url, json={
    "chat_id": CHAT_ID,
    "text": "안녕하세요"
}, timeout=15)
requests.get과 requests.post는 각각 https://api.telegram.org/bot<토큰>/METHOD_NAME 주소로 인터넷 요청을 보냅니다. 텔레그램 서버는 토큰으로 어느 봇의 요청인지 확인한 뒤, getUpdates는 쌓인 메시지 목록을, sendMessage는 발송 성공 여부를 응답으로 돌려줍니다.
토큰을 코드 안에 그대로 적어 GitHub 같은 공개 저장소에 올리지 않도록 주의합니다. Day 5에서 GitHub Actions로 자동화할 때는 토큰을 코드가 아닌 별도의 비밀 값(secret) 저장소에 넣는 방법을 다룹니다.
● 고급 Power User 받은 메시지 내용에 따라 다르게 응답하는 간단한 명령어 처리 로직을 설계합니다
P-1 고급 Claude Code와 함께 /start, /status 명령어에 응답하는 로직 설계하기
지금까지는 봇이 항상 같은 메시지만 보냈습니다. 이번에는 사용자가 /start나 /status 같은 명령어를 보냈을 때, 봇이 그 내용을 구분해서 서로 다르게 응답하도록 Claude Code와 함께 로직을 설계하려고 합니다. 이는 Day 4~5에서 만들 뉴스 알림 봇에도 그대로 응용됩니다.
사용 요소
getUpdates sendMessage offset
단계별 실행
  1. Claude Code에게 요청 내용 설명하기

    "getUpdates로 받은 메시지의 text가 /start인지 /status인지 확인해서, 각각 다른 안내 문구를 보내는 파이썬 함수를 짜줘. 그리고 한 번 처리한 메시지를 중복해서 또 처리하지 않도록 offset도 관리해줘"라고 요청합니다.

  2. 명령어 분기 로직 확인하기

    Claude Code가 아래와 비슷한 형태의 코드를 만들어 줍니다. tg_get은 Bot API의 GET 메서드를, send_message는 sendMessage를 감싼 보조 함수이고, offset은 이미 처리한 메시지의 update_id 바로 다음 값으로 갱신해 같은 메시지를 두 번 처리하지 않게 합니다.

    import requests

TOKEN = "<여기에_발급받은_BOT_TOKEN>"
TGAPI = f"https://api.telegram.org/bot{TOKEN}"

def tg_get(method, **params):
    r = requests.get(f"{TGAPI}/{method}", params=params, timeout=15)
    r.raise_for_status()
    return r.json()

def send_message(chat_id, text):
    requests.post(f"{TGAPI}/sendMessage", json={"chat_id": chat_id, "text": text}, timeout=15)

offset = None

def handle_updates():
    global offset
    resp = tg_get("getUpdates", offset=offset, timeout=0)
    for upd in resp.get("result", []):
        offset = upd["update_id"] + 1
        msg = upd.get("message", {})
        text = msg.get("text", "").strip()
        chat_id = msg.get("chat", {}).get("id")

        if text == "/start":
            send_message(chat_id, "안녕하세요! 뉴스 알림 봇입니다.")
        elif text == "/status":
            send_message(chat_id, "정상 작동 중입니다. 뉴스가 검색되면 알려드립니다.")
  3. 명령어 추가로 확장해 보기

    같은 방식으로 elif 구문을 추가하면 /help 같은 명령어도 얼마든지 늘릴 수 있습니다. Claude Code에게 "명령어를 하나 더 추가해줘"라고 요청하면 같은 패턴으로 확장해 줍니다.

getUpdates가 반환하는 각 항목에는 고유한 update_id가 붙어 있습니다. 처리할 때마다 offset을 그 값+1로 올려두면, 다음 호출에서는 이미 처리한 메시지를 다시 받지 않습니다. 이 반복 호출 구조가 이후 GitHub Actions의 정기 실행(cron)과 결합되면, 사람이 지켜보지 않아도 새 메시지가 올 때마다 자동으로 응답하는 봇이 됩니다.
지금은 코드를 직접 실행해야만 봇이 응답하지만, Day 5에서 GitHub Actions로 이 로직을 주기적으로 자동 실행하도록 설정하면 사람이 켜 두지 않아도 항상 대기 중인 봇이 됩니다.
