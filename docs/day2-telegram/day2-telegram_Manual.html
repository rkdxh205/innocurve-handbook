1. 텔레그램이란, 왜 텔레그램으로 알림을 받는가

Telegram — 전 세계에서 널리 쓰이는 메신저이자 개방형 봇 플랫폼

텔레그램은 카카오톡과 비슷한 메신저 앱이지만, 처음부터 "봇(bot)"을 자유롭게 만들어 붙일 수 있도록 설계된 것이 큰 차이점입니다. 누구나 무료로 계정을 만들고, 자신만의 자동화 프로그램을 봇 형태로 연결해 메시지를 주고받을 수 있습니다. 이번 핸드북에서는 이 봇을 "내 이름이 언급된 뉴스를 알려주는 개인 비서"로 활용합니다.

Bot API — 텔레그램이 공식 제공하는 봇 프로그래밍 인터페이스

Telegram Bot API는 별도의 서버나 복잡한 설정 없이도, 인터넷 요청 한 번으로 메시지를 보내고 받을 수 있게 해주는 공식 인터페이스입니다. 로그인 절차나 승인 심사가 없고, 봇 하나당 발급되는 "토큰" 하나만 있으면 바로 사용할 수 있어 진입 장벽이 매우 낮습니다.

💡 핵심 한 줄: 텔레그램은 API가 공개돼 있고 무료이며, GitHub Actions 같은 자동화 도구와도 궁합이 좋아 "개인 알림 채널"로 안성맞춤입니다.

비교 항목텔레그램카카오톡·문자(SMS)
봇 API 공개 여부 공식적으로 완전히 공개, 무료 개인 개발자용 공개 API가 제한적이거나 유료
알림 자동화 난이도 토큰 하나로 즉시 메시지 발송 가능 별도 심사·계약이 필요한 경우가 많음
이번 핸드북과의 궁합 GitHub Actions의 정기 실행 결과를 그대로 전달하기 적합 해당 없음

2. BotFather로 봇 만들기

@BotFather — 모든 텔레그램 봇을 만들고 관리하는 공식 봇

BotFather는 텔레그램이 공식으로 운영하는 "봇을 만들어주는 봇"입니다. 새 봇을 등록하거나, 이름을 바꾸거나, 프로필 사진을 설정하는 등 봇 관리에 필요한 모든 명령을 대화 형태로 처리합니다.

  1. @BotFather 검색: 텔레그램 앱 상단 검색창에 @BotFather를 입력해 공식 계정(파란 인증 배지가 있는 계정)을 찾습니다.
  2. 대화 시작: BotFather와의 대화방에 들어가 시작(Start) 버튼을 눌러 대화를 엽니다.
  3. /newbot 전송: 새 봇을 만들겠다는 명령을 보냅니다.
  4. 표시 이름 입력: 대화 목록 등에 보일 봇의 이름을 입력합니다. (예: "홍길동 뉴스 알림봇" — 한글도 가능하며 나중에 변경할 수 있습니다.)
  5. username 입력: 봇의 고유 주소가 되는 username을 입력합니다. 아래 규칙을 따라야 하며, 한 번 정하면 바꿀 수 없습니다.

username 규칙 — 한 번 정하면 바꿀 수 없는 봇의 고유 주소

BotFather는 입력한 username이 규칙에 맞고 아직 아무도 쓰지 않은 이름일 때만 봇 생성을 완료합니다.

규칙내용비고
길이 5자 이상 32자 이하 너무 짧으면 거부됩니다
허용 문자 영문 알파벳, 숫자, 밑줄(_)만 가능 한글·공백·특수문자 불가
끝 글자 반드시 bot으로 끝나야 함 예: gildong_news_bot
변경 가능 여부 한 번 정하면 이후 변경 불가 신중하게 정하는 것을 권장합니다
username은 전 세계 사용자와 공유하는 이름 공간이라 이미 다른 사람이 쓰고 있으면 등록되지 않습니다. 여러 후보를 미리 생각해 두는 것이 좋습니다.

3. 봇 토큰이란 무엇인가

Bot Token — 봇의 신원과 권한을 증명하는 비밀 문자열

username 등록이 끝나면 BotFather는 곧바로 "봇 토큰(token)"을 발급해 대화창에 보여줍니다. 이 토큰은 앞으로 프로그램이 "나는 이 봇이다"라고 증명할 때 쓰는 열쇠이며, 토큰만 있으면 누구든 그 봇의 이름으로 메시지를 보내거나 받을 수 있습니다.

토큰은 다음과 같은 형식으로 발급됩니다.

110201543:AAHdqTcvCH1vGWJxfSeofSAs0K5PALDsaw

앞부분의 숫자(110201543)는 봇의 고유 ID이고, 콜론(:) 뒤의 약 35자 문자열이 실제 비밀 값입니다. (※ 위 예시는 텔레그램 공식 튜토리얼에 나오는 예시 형식이며, 실제로 발급받은 토큰은 이와 다른 값입니다.)

봇 토큰은 집 열쇠나 비밀번호와 동일하게 취급해야 합니다. GitHub 등 공개 저장소 코드에 그대로 적어 올리거나, 화면 공유·발표 중에 화면에 노출하지 않도록 각별히 주의하세요. 토큰이 유출되면 즉시 BotFather에서 /revoke 명령으로 재발급받아야 합니다.

💡 핵심 한 줄: 발급받은 토큰은 메모장 등에 안전하게 적어두고, 이후 실습(Day 3~5)에서 코드나 GitHub Actions의 비밀 값(secret)으로 사용하게 됩니다.

4. Telegram Bot API 기초

모든 Bot API 호출은 아래와 같이 토큰을 포함한 하나의 주소(URL) 형태를 따릅니다. 이 주소에 필요한 값(파라미터)을 담아 요청을 보내면, 텔레그램 서버가 그 결과를 돌려줍니다.

https://api.telegram.org/bot<여기에_발급받은_BOT_TOKEN>/METHOD_NAME

HTTPS GET 또는 POST 토큰 필요

sendMessage — 봇이 특정 대화방에 메시지를 보내는 메서드

가장 기본적인 메서드로, 어느 대화방(chat_id)에 어떤 내용(text)을 보낼지 지정하면 그대로 메시지가 전송됩니다. 이번 핸드북의 최종 목표인 "뉴스 알림 보내기"의 핵심 동작이 바로 이 메서드입니다.

파라미터필수 여부설명
chat_id 필수 메시지를 받을 대화방의 고유 번호
text 필수 보낼 메시지 내용
parse_mode 선택 굵게·기울임 등 서식을 적용할지 지정 (예: Markdown, HTML)

getUpdates — 롱 폴링(long polling)으로 대기 중인 메시지를 가져오는 메서드

봇에게 도착했지만 아직 프로그램이 확인하지 않은 메시지들을 가져오는 메서드입니다. "롱 폴링"이란 서버에 접속해서 새 메시지가 올 때까지 짧게 기다렸다가 응답받는 방식으로, 별도의 웹 서버 없이도 메시지 수신이 가능하게 해줍니다.

파라미터기본값설명
offset 없음 이 번호 이후의 새 메시지만 가져오도록 지정 (이미 처리한 메시지를 다시 받지 않기 위함)
limit 100 한 번에 가져올 메시지 최대 개수 (1~100 사이)
timeout 0 새 메시지가 없을 때 몇 초 동안 대기할지 (초 단위)
allowed_updates 모든 종류 받고 싶은 업데이트 종류를 제한 (예: 일반 메시지만)
봇에 웹훅(webhook)이 설정돼 있으면 getUpdates는 동작하지 않습니다. 이번 핸드북에서는 웹훅을 설정하지 않으므로 문제없이 사용할 수 있습니다.

chat_id 알아내기 — 내 대화방 번호를 확인하는 표준 절차

메시지를 보내려면 상대방(자기 자신)의 chat_id를 알아야 합니다. 아래 절차를 따르면 누구나 자신의 chat_id를 쉽게 확인할 수 있습니다.

  1. 봇에게 말 걸기: 텔레그램에서 방금 만든 내 봇을 검색해 대화방을 열고, 아무 텍스트나 한 번 보냅니다. (예: "안녕")
  2. getUpdates 호출: 발급받은 토큰으로 getUpdates 메서드를 호출합니다.
  3. chat.id 확인: 응답 결과(JSON) 안의 message.chat.id 값이 바로 내 chat_id입니다. 이 값을 메모해 두면 Day 3부터 계속 사용하게 됩니다.
