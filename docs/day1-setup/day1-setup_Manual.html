1. 생성형 AI와 Claude란 무엇인가

LLM — "정답을 아는 존재"가 아니라 "그럴듯한 다음 단어를 예측하는 존재"

Claude와 같은 대규모 언어 모델(LLM, Large Language Model)은 방대한 글을 학습해, 주어진 문맥 다음에 올 확률이 가장 높은 단어를 순서대로 이어 붙이는 방식으로 문장을 만듭니다. 즉 "이 질문의 정답이 데이터베이스에 저장돼 있어서 그것을 꺼내 오는 것"이 아니라, "지금까지의 대화 맥락을 보고 가장 그럴듯한 다음 문장을 매번 새로 생성하는 것"에 가깝습니다. 이 원리를 이해하면 왜 같은 질문에도 답이 조금씩 달라질 수 있는지, 왜 맥락(지시문, 예시, 배경 설명)을 충분히 주는 것이 결과물의 품질을 좌우하는지를 자연스럽게 받아들일 수 있습니다.

context — 맥락을 주는 만큼 결과물이 좋아진다

Claude는 대화창에 입력된 내용, 첨부된 파일, 이전 대화 기록 등 "맥락"을 근거로 답을 생성합니다. "뉴스 기사를 요약해줘"보다 "부총리·차관급 정책 담당자가 5분 내로 훑어볼 수 있도록, 핵심 쟁점과 숫자 위주로 6줄 이내로 요약해줘"처럼 맥락을 구체적으로 줄수록 실무에 바로 쓸 수 있는 결과가 나옵니다. 이번 5일 과정에서도 Claude에게 코드 작성을 요청할 때마다 "무엇을, 왜, 어떤 형태로" 원하는지 맥락을 명시하는 습관이 중요합니다.

hallucination — Claude가 잘하는 일과 사람이 반드시 판단해야 하는 일

Claude는 반복적인 코드 작성, 문서 초안 작성, 오류 메시지 해석, 절차 설명 같은 작업에 매우 능숙합니다. 반면 존재하지 않는 사실을 그럴듯하게 지어내는 "환각(hallucination)" 현상이 있을 수 있으므로, 최종 정책 판단·법적 책임이 따르는 의사결정·외부에 실제로 발송되는 메시지 내용의 최종 확인은 반드시 담당자가 직접 검토해야 합니다. 이번 과정에서 만들 뉴스 알림 봇도 "기사를 찾아 알려주는 역할"까지는 자동화하되, 그 내용을 바탕으로 한 판단은 담당자의 몫으로 남겨 둡니다.

구분Claude가 잘하는 일사람이 반드시 판단해야 하는 일
코드/자동화 반복적인 코드 작성, 오류 원인 설명, 절차 안내 실제 배포 여부, 민감 정보 취급 방식 최종 승인
문서/텍스트 초안 작성, 요약, 번역, 형식 변환 사실관계 최종 검증, 정책적 표현 수위 결정
의사결정 선택지 정리, 장단점 비교 최종 선택과 그에 따른 책임

💡 핵심 한 줄: Claude는 맥락을 주면 그럴듯한 다음 문장을 만들어내는 도구이지, 정답을 알고 있는 존재가 아닙니다 — 최종 판단은 언제나 사람의 몫입니다.

2. claude.ai 계정 가입

claude.ai — 웹 브라우저에서 바로 시작하는 가장 쉬운 방법

claude.ai에 접속해 "Continue with email" 또는 "Continue with Google" 버튼을 클릭하면 계정을 만들 수 있습니다. 이메일 방식을 선택하면 로그인 링크가 담긴 메일이 오는데, 메일을 받은 것과 같은 기기에서 열면 자동으로 로그인되고, 다른 기기에서 열면 화면에 인증 코드를 입력해야 합니다. 신규 계정은 전화번호 인증(국가번호 선택 후 번호 입력)과 만 18세 이상 확인 절차를 거치며, 이어서 SMS로 전송되는 6자리 코드를 입력해야 가입이 완료됩니다.

  1. claude.ai 접속: 브라우저 주소창에 claude.ai를 입력해 접속합니다.
  2. 로그인 방식 선택: "Continue with email" 또는 "Continue with Google" 중 하나를 클릭합니다.
  3. 이메일 인증: 이메일 방식이라면 받은 메일의 로그인 링크를 클릭하거나, 다른 기기라면 화면에 표시된 인증 코드를 입력합니다.
  4. 전화번호 인증(신규 계정): 국가번호와 전화번호를 입력하고, 만 18세 이상임을 확인합니다.
  5. SMS 코드 확인: 문자로 받은 6자리 코드를 입력하면 가입이 완료됩니다.

Free / Pro — 무료 플랜과 Pro 플랜 비교

가입 직후에는 Free 플랜으로 시작하며, 웹·iOS·Android·Desktop에서 채팅 형태로 Claude를 바로 이용할 수 있습니다. 이번 5일 과정의 핵심 도구인 Claude Code(에디터 안에서 실제 코드 파일을 만들고 수정하는 기능)를 쓰려면 Pro 플랜 이상이 필요하므로, Day 1에서 업그레이드 여부를 미리 확인해 두는 것이 좋습니다.

upgrade — Pro 플랜으로 업그레이드하는 경로

Pro 플랜은 로그인 후 화면 좌하단의 이름을 클릭해 들어가는 설정 메뉴에서 신청할 수 있습니다. 아래 단계를 그대로 따라가면 됩니다.

  1. Settings 진입: 화면 좌하단 이름을 클릭한 뒤 Settings를 선택합니다.
  2. Billing 메뉴: Settings 안에서 Billing 탭으로 이동합니다.
  3. 업그레이드 시작: Upgrade plan 버튼을 클릭한 뒤 Get Pro plan을 선택합니다.
  4. 결제 주기 선택: 월간 결제와 연간 결제 중 원하는 방식을 고릅니다.
  5. 결제 정보 입력 및 완료: 결제 정보를 입력하고 Subscribe를 눌러 구독을 완료합니다.

3. Claude Desktop 설치

Claude Desktop — 컴퓨터에 상주하는 별도 앱

Claude Desktop은 브라우저를 열지 않고도 바탕화면에서 바로 Claude와 대화할 수 있는 전용 앱입니다. 공식 다운로드 페이지인 claude.com/download에서 사용 중인 운영체제에 맞는 설치파일을 내려받을 수 있습니다.

운영체제지원 버전비고
macOS macOS 11 이상 Desktop 앱 정식 지원
Windows Windows 10 이상 Desktop 앱 정식 지원
Linux 주요 배포판 지원 베타 지원 — 배포판별 요건은 다운로드 페이지에서 확인
iOS / Android 모바일 최신 버전 별도 모바일 앱으로 제공

install — 설치 절차

운영체제만 정확히 확인하면 설치 자체는 몇 번의 클릭으로 끝납니다. Windows 기준으로 다음 순서를 따르면 됩니다.

  1. 다운로드 페이지 접속: claude.com/download에 접속합니다.
  2. OS에 맞는 버튼 클릭: Windows(또는 macOS, Linux) 버튼을 클릭해 설치파일을 내려받습니다.
  3. 설치파일 실행: 내려받은 설치파일을 더블클릭해 실행하고, 안내에 따라 설치를 완료합니다.
  4. 로그인: 설치가 끝나면 Claude Desktop을 실행해, 앞서 만든 claude.ai 계정으로 로그인합니다.
설치 후 로그인이 안 되거나 화면이 비어 보이면, 사내 방화벽이나 보안 프로그램이 연결을 차단하고 있는 경우가 많습니다. 이런 경우 담당 부서에 claude.ai 및 관련 도메인 접속 허용을 요청해야 합니다.

4. Cursor 에디터란 무엇이고 왜 쓰는가

Cursor — VS Code 기반의 AI 코딩 에디터

Cursor는 개발자들이 널리 쓰는 무료 에디터 VS Code(Visual Studio Code)를 기반으로 만들어진 코드 편집기입니다. 겉모습과 기본 조작 방식이 VS Code와 거의 같아서, VS Code용으로 만들어진 대부분의 기능(확장)을 그대로 사용할 수 있다는 장점이 있습니다. 이번 과정에서는 Cursor 위에 Claude Code 확장을 설치해, 코드 편집 화면과 AI 대화 화면을 하나의 창 안에서 오가며 실습을 진행합니다.

extension — 확장(익스텐션)이란 무엇인가

확장(Extension)은 에디터에 새로운 기능을 덧붙이는 일종의 "부속 프로그램"입니다. 스마트폰에 앱을 설치해 기능을 늘리는 것과 비슷하게, Cursor에도 원하는 확장을 검색해서 설치하면 없던 기능이 새로 생깁니다. 이번 과정에서 설치할 Claude Code 확장이 대표적인 예로, 이 확장을 설치하면 Cursor 안에서 Claude에게 직접 코드 작업을 맡길 수 있게 됩니다.

marketplace — 확장을 어디서 받아오는가

Cursor는 두 가지 마켓플레이스(확장 저장소)를 사용합니다. 하나는 Open VSX 마켓플레이스이고, 다른 하나는 Cursor 자체 마켓플레이스(cursor.com/marketplace)입니다. 실습에서는 Cursor 안의 Extensions 뷰에서 검색창에 이름만 입력하면 되므로, 마켓플레이스 주소를 직접 방문할 필요는 없습니다.

항목설명비고
다운로드 cursor.com/download에서 운영체제별 설치파일 제공 Mac / Windows / Linux
Extensions 뷰 단축키 Windows/Linux는 Ctrl+Shift+X, Mac은 Cmd+Shift+X 좌측 사이드바에서 아이콘 클릭으로도 진입 가능
설치 버튼 검색 결과에서 원하는 확장을 선택 후 Install 클릭 설치 완료까지 몇 초 소요

5. Claude Code 확장 설치 및 연결

anthropic.claude-code — Cursor 안에서 Claude에게 코드 작업을 맡기는 확장

Claude Code 확장(공식 명칭 "Claude Code for VS Code", 확장 ID anthropic.claude-code)은 Cursor(또는 VS Code) 안에 대화창을 띄워, 그 대화창에서 Claude에게 실제로 파일을 만들거나 코드를 수정하도록 요청할 수 있게 해주는 확장입니다. 지금까지는 claude.ai나 Claude Desktop에서 "대화"만 할 수 있었다면, 이 확장을 설치한 뒤부터는 Claude가 직접 프로젝트 폴더 안의 파일을 읽고 쓰는 실질적인 작업까지 수행할 수 있습니다. 이번 5일 과정에서 만들 뉴스 알림 봇의 실제 코드도 모두 이 확장을 통해 작성하게 됩니다.

install — 확장 설치 절차

  1. Extensions 뷰 열기: Cursor에서 Ctrl+Shift+X(Windows/Linux) 또는 Cmd+Shift+X(Mac)를 눌러 Extensions 뷰를 엽니다.
  2. 검색: 검색창에 "Claude Code"를 입력합니다.
  3. 설치: 검색 결과 중 "Claude Code for VS Code"(게시자 Anthropic)를 찾아 Install 버튼을 클릭합니다.

login — 최초 로그인 및 연결

확장을 설치한 뒤 처음 실행하면 로그인 절차가 필요합니다. 브라우저 창이 자동으로 열리면서 OAuth(웹 기반 인증) 방식으로 로그인이 진행되며, Pro·Max·Team·Enterprise 플랜 중 하나 또는 Console 계정만 있으면 되고 별도의 API 키를 발급받을 필요는 없습니다.

  1. 확장 실행: Cursor 좌측 사이드바에서 Claude Code 아이콘을 클릭해 대화창을 엽니다.
  2. 로그인 요청 확인: 처음 실행 시 로그인이 필요하다는 안내가 표시됩니다.
  3. 브라우저 인증: 자동으로 열리는 브라우저 창에서 claude.ai 계정으로 로그인해 인증을 완료합니다.
  4. 연결 확인: Cursor로 돌아와 대화창에 정상적으로 응답이 오는지 확인합니다.
Free 플랜 계정으로는 Claude Code 확장을 사용할 수 없습니다. 로그인이 계속 실패한다면 먼저 계정이 Pro 플랜 이상으로 업그레이드되어 있는지(섹션 2 참고) 확인해야 합니다.

💡 핵심 한 줄: Claude Code 확장이 연결되면, 그때부터 Cursor는 단순한 편집기가 아니라 "AI에게 실제 파일 작업을 시킬 수 있는 작업대"가 됩니다.

6. PRD 작성 — 만들기 전에 먼저 계획하기

PRD — 코드보다 먼저 쓰는 한 장의 설계도

PRD(Product Requirements Document, 제품요구사항정의서)는 "무엇을, 누구를 위해, 왜 만드는가"를 코드를 쓰기 전에 미리 정리해두는 문서입니다. 화려한 기획서일 필요는 없습니다 — Day 2부터 시작할 실습이 방향을 잃지 않도록 잡아주는 나침반 한 장이면 충분합니다. 도구 준비를 마친 오늘, Claude와 대화하며 이번 5일간 만들 "내 이름 뉴스 알림 봇"의 PRD를 직접 작성해봅니다.

Who / What / Why — PRD를 시작하는 세 가지 질문

모든 PRD는 이 세 질문에서 출발합니다: 누가 쓰는가(Who), 무엇을 하는가(What), 왜 필요한가(Why). 이번 프로젝트에 대입하면, Who는 나 자신(텔레그램으로 알림을 받는 1인 사용자), What은 "내 이름이 들어간 뉴스를 자동으로 찾아 알려주는 것", Why는 "매번 직접 검색하는 반복 작업을 줄이기 위해서"가 됩니다. 이 세 문장만 먼저 명확히 해두면, 나머지 항목은 Claude와의 대화로 빠르게 채울 수 있습니다.

MoSCoW — 기능의 우선순위를 나누는 법

처음부터 모든 기능을 다 만들려고 하면 5일 안에 끝내기 어렵습니다. MoSCoW는 기능을 네 등급으로 나눠 "이번에 반드시 만들 것"과 "나중으로 미룰 것"을 구분하는 방법입니다. Day 2~4에서는 Must에 해당하는 기능만 먼저 완성하고, Should·Could는 시간이 남으면 도전하는 순서로 진행합니다.

등급의미이번 프로젝트 예시
Must 이것 없이는 서비스가 성립하지 않는 필수 기능 뉴스 검색, 텔레그램 전송, 하루 2회 자동 실행
Should 있으면 크게 도움되지만 없어도 동작은 하는 기능 동명이인 필터링, 중복 기사 제거
Could 여유가 있으면 추가하는 기능 구독/구독취소 명령어, 오류 알림
Won't 이번 5일 안에는 하지 않기로 정한 기능 여러 사람의 이름을 동시에 검색하는 기능, 웹 화면 제공

구성요소 — PRD 한 장에 담을 항목

아래 표의 항목을 Claude에게 순서대로 채워달라고 요청하면, 짧은 대화만으로도 실전에 쓸 수 있는 PRD 초안이 완성됩니다. 이 항목들은 이번 실습(Examples 문서의 P-1 카드)에서 그대로 사용합니다.

항목담을 내용
개요 및 목적 서비스명, 한 줄 설명, 왜 지금 필요한가
대상 사용자와 니즈 누가 쓰는가(Who), 그 사람이 겪는 불편(What/Why)
핵심 기능 목록 기능명·설명·MoSCoW 우선순위
화면·동작 흐름 무엇을 입력하면(예: 검색어) 무엇이 출력되는가(예: 텔레그램 메시지)
성공지표(KPI) 목표를 달성했는지 확인할 정량 지표(예: 하루 2회 정상 실행 여부)
제외 범위 이번 5일 안에는 하지 않을 것과 그 이유

💡 핵심 한 줄: 완벽한 PRD를 쓰는 것이 목표가 아닙니다 — Day 2 실습을 망설임 없이 시작할 수 있는 최소한의 설계도면 충분합니다.

PRD는 한 번 쓰고 끝나는 문서가 아닙니다. Day 3~4를 진행하며 생각이 바뀌면 언제든 Claude에게 "PRD의 이 항목을 이렇게 수정해줘"라고 요청해 계속 다듬어 나가면 됩니다.
